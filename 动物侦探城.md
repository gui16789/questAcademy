## 1. 项目概述

### 1.1 项目名称

**动物侦探城：徽章失踪案**

### 1.2 项目定位

面向小学二年级下册学生的数学游戏化学习 MVP。

通过“动物城市探案 + 知识点闯关 + 线索收集 + Boss 挑战 + 错题悬案馆 + 知识线索库 + 勋章激励”的方式，让孩子在轻松的探案故事中完成北师大版数学二年级下册第一单元“除法”的学习与练习。

### 1.3 MVP 目标

MVP 不追求完整大世界和全册内容，而是验证一个核心问题：

> 小学二年级孩子是否愿意通过探案闯关的方式完成数学学习闭环，并在错题复习和知识点掌握上产生正向反馈。

### 1.4 核心学习闭环

```
进入动物侦探城
→ 接到案件
→ 阅读案件背景
→ 完成普通知识点关卡
→ 答题获得线索
→ 错题进入悬案馆
→ 集齐线索解锁 Boss
→ 挑战 Boss 综合题
→ 完成结案
→ 获得勋章
→ 进入线索库复习知识点
→ 进入悬案馆重做错题
```

---

## 2. 背景与问题

### 2.1 当前问题

小学低年级数学练习常见问题：

- 题目重复，孩子容易觉得枯燥。
- 错题复习缺乏趣味，孩子抗拒回看。
- 知识点之间缺少故事化关联。
- 家长很难判断孩子到底是“不认真”还是“知识点没掌握”。
- 普通刷题产品容易只强调对错，忽略坚持、复盘和阶段成就感。

### 2.2 产品机会

将数学练习包装为“动物城市探案流程”：

- 普通知识点练习 = 搜集线索
- 错题库 = 悬案馆
- 知识库 = 线索库
- 综合应用题 = Boss 挑战
- 学习成就 = 勋章记录
- 完成学习单元 = 破案结案

让孩子把学习过程理解成“破案”，而不是“被动刷题”。

## 3. 目标用户

### 3.1 核心用户

```
小学二年级下册学生，年龄约 7～8 岁。
```

### 3.2 使用场景

```
家庭课后练习为主。
孩子可以独立操作，家长可以旁观或轻度辅助。
单次使用时长控制在 10～15 分钟。
```

### 3.3 用户特点

- 已具备基础识字能力，但长文本阅读能力有限。
- 对故事、动物角色、闯关奖励有较高兴趣。
- 容易因为连续做错题产生挫败感。
- 更适合短流程、强反馈、低压力的学习体验。

### 3.4 家长期望

- 希望孩子愿意主动练习。
- 希望练习内容和教材同步。
- 希望能看到孩子错在哪里。
- 希望错题能够复习，而不是做完就结束。
- 希望产品不要过度游戏化、分散学习注意力。

## 4. MVP 范围

### 4.1 教材范围

```
教材版本：北师大版小学数学二年级下册
首期单元：第一单元“除法”
```

### 4.2 首期知识点

MVP 首期围绕“除法 / 有余数除法”展开，建议包含以下知识点：

| 序号  | 知识点      | 学习目标             |
| --- | -------- | ---------------- |
| 1   | 平均分      | 理解把物品平均分成若干份     |
| 2   | 除法算式含义   | 能根据平均分情境写出除法算式   |

| 4   | 余数小于除数   | 理解余数必须比除数小       |
| 5   | 除法解决实际问题 | 能在生活情境中判断商和余数的意义 |

## 5. IP 与美术原则

### 5.1 IP 原则

MVP 使用原创动物城市设定，不直接使用“疯狂动物城”的角色、名称、地图、画面或剧情。参考进行单独设计。

### 5.2 原创世界观

暂定世界观：

```
动物侦探城是一座由各种动物共同生活的城市。
城市中有一所“动物侦探学院”，孩子扮演刚入学的新手小侦探。
每一次学习任务都会包装成一个案件。
孩子通过数学调查、线索收集和推理挑战完成破案。
```

### 5.3 美术风格

| 项目   | 风格要求          |
| ---- | ------------- |
| 整体风格 | 可爱、明亮、轻探案、低压力 |
| 色彩   | 温暖、活泼，避免阴暗恐怖  |
| 角色   | 原创动物角色        |
| 地图   | 简化动物城市地图      |
| 图标   | 清晰易懂，适合儿童     |
| 勋章   | 具有收集感和奖励感     |

## 6. 案件设计

### 6.1 首个案件名称

```
徽章失踪案
```

### 6.2 案件背景

动物侦探学院准备给新手小侦探们发放“胡萝卜徽章”。可是，在正式分发前，仓库管理员发现徽章数量对不上。

孩子需要作为新手小侦探，调查仓库、分发记录、运输路线和嫌疑动物的说法，通过除法知识找出徽章失踪的真相。

### 6.3 案件目标

```
通过学习和练习除法相关知识，收集 4 条关键线索，解锁 Boss 挑战，最终找出徽章数量异常的原因并完成结案。
```

### 6.4 案件角色

| 角色    | 说明            |
| ----- | ------------- |
| 小侦探   | 孩子扮演的主角       |
| 鹿队长   | 引导孩子接案、学习和推理  |
| 兔仓管   | 负责保管徽章，提供仓库线索 |
| 狐狸配送员 | 负责运输徽章，提供分发线索 |
| 熊老师   | 负责颁发徽章，提供最终记录 |

### 6.5 案件结构

| 阶段    | 内容       | 对应学习目标    |
| ----- | -------- | --------- |
| 接案    | 了解徽章失踪背景 | 建立任务目标    |
| 第 1 关 | 平均分徽章    | 理解平均分     |
| 第 2 关 | 查看分发记录   | 理解除法算式    |
| 第 3 关 | 查找剩余徽章   | 理解有余数除法   |
| 第 4 关 | 判断嫌疑说法   | 理解余数小于除数  |
| Boss  | 综合推理     | 用除法解决实际问题 |
| 结案    | 揭示真相     | 完成学习闭环    |

## 7. 核心流程

### 7.1 用户主流程

```
打开产品
→ 进入开始页
→ 输入昵称或选择游客身份
→ 进入动物侦探城地图
→ 点击“徽章失踪案”
→ 阅读案件背景
→ 开始第 1 个普通关卡
→ 完成答题
→ 获得线索
→ 依次完成 4 个普通关卡
→ 集齐 4 条线索
→ 解锁 Boss 挑战
→ 完成 Boss
→ 查看案件真相
→ 获得勋章
→ 返回地图
→ 可进入悬案馆或线索库复习
```

### 7.2 答题流程

```
进入关卡
→ 展示关卡剧情
→ 展示题目
→ 用户作答
→ 系统判断对错
→ 答对：获得正向反馈
→ 答错：展示温和提示，并加入悬案馆
→ 完成本关题目
→ 判断是否通过
→ 通过后获得线索
```

### 7.3 错题复习流程

```
进入悬案馆
→ 查看未侦破错题
→ 选择一道错题重新调查
→ 再次作答
→ 答对：标记为已侦破
→ 答错：保持未侦破状态，并可查看解析
```

### 7.4 知识复习流程

```
进入线索库
→ 查看已获得知识线索卡
→ 选择知识点
→ 查看一句话解释、例题、方法和常见错误
→ 可返回对应关卡或悬案馆复习
```

## 8.4 普通关卡

### 功能说明

普通关卡承载单个知识点练习。

### 关卡设计

| 关卡    | 名称    | 知识点    | 题目数量 | 通过条件     |
| ----- | ----- | ------ | ---- | -------- |
| 第 1 关 | 仓库平均分 | 平均分    | 4 道  | 答对至少 3 道 |
| 第 2 关 | 分发记录表 | 除法算式含义 | 4 道  | 答对至少 3 道 |
| 第 3 关 | 剩下的徽章 | 有余数除法  | 4 道  | 答对至少 3 道 |
| 第 4 关 | 可疑的说法 | 余数小于除数 | 4 道  | 答对至少 3 道 |

### 答题规则

- 每关 4 道题。
- 答对 3 道及以上通过。
- 答错不阻断流程。
- 答错题进入悬案馆。
- 每通过一关获得 1 条线索。
- 4 个普通关卡全部通过后解锁 Boss。

### 验收标准

- 系统能展示题目并判断答案。
- 系统能统计本关答对数量。
- 系统能判断是否通过。
- 答错题会进入悬案馆。
- 通过关卡后发放对应线索。

---

## 8.5 线索获得页

### 功能说明

用户通过普通关卡后展示本关获得的线索。

### 线索示例

| 关卡    | 获得线索               |
| ----- | ------------------ |
| 第 1 关 | 仓库里的徽章原本可以平均分给几个小队 |
| 第 2 关 | 分发记录中有一组除法算式不对     |
| 第 3 关 | 剩下的徽章数量是一条重要线索     |
| 第 4 关 | 有个嫌疑动物说的余数不合理      |

### 验收标准

- 用户通过关卡后能看到线索。
- 线索会保存到案件进度中。
- 线索对应的知识卡会进入线索库。

---

## 8.6 Boss 挑战

### 功能说明

Boss 是综合应用关卡，用于检验多个知识点的串联使用。

### Boss 规则

- 收集 4 条线索后解锁。
- Boss 包含 2 道综合应用题。
- Boss 题覆盖“平均分、除法算式、有余数除法、余数小于除数、实际问题”。
- Boss 答错题进入悬案馆。
- Boss 失败后可以重试，也可以返回线索库复习。
- Boss 通过后进入结案页。

### Boss 示例

```
动物侦探学院原来有 35 枚胡萝卜徽章。
每个侦探小队分 6 枚。
分完后可以分给几个小队？还剩几枚？

如果狐狸配送员说“剩下的徽章还能再分给一个完整小队”，这句话对吗？
```

### 验收标准

- 未集齐线索时 Boss 不可进入。
- 集齐线索后 Boss 自动解锁。
- Boss 答题结果会被记录。
- Boss 错题会进入悬案馆。
- Boss 通过后案件状态变为已结案。

---

## 8.7 结案页

### 功能说明

展示案件真相、学习总结和奖励。

### 页面元素

- 案件真相
- 本次学习完成情况
- 已掌握知识点
- 本次新增错题数量
- 获得勋章
- 返回地图按钮
- 去线索库复习按钮
- 去悬案馆调查按钮

### 验收标准

- Boss 通过后自动进入结案页。
- 结案页展示案件结果。
- 系统发放“破案小能手”勋章。
- 用户可以从结案页进入线索库或悬案馆。

---

## 8.8 悬案馆

### 功能说明

悬案馆是错题库，用游戏化语言包装错题复习。

### 错题进入规则

以下情况题目进入悬案馆：

- 普通关卡答错
- Boss 挑战答错

### 错题状态

| 状态  | 含义        |
| --- | --------- |
| 未侦破 | 做错后进入悬案馆  |
| 已侦破 | 重新答对后标记完成 |

### 错题展示内容

- 题目内容
- 孩子原答案
- 正确答案
- 儿童化解析
- 所属知识点
- 错误次数
- 最近错误时间
- 当前状态

### 解除规则

```
错题重新答对一次后，状态变为“已侦破”。
```

### 验收标准

- 答错题会自动加入悬案馆。
- 悬案馆能展示未侦破和已侦破错题。
- 用户可以重新作答错题。
- 重新答对后状态更新为已侦破。
- 悬案馆数据刷新后仍然保留。

---

## 8.9 线索库

### 功能说明

线索库是知识点复习库，用于沉淀已经学过的数学知识。

### 知识卡内容

每张知识线索卡包含：

- 知识点名称
- 一句话解释
- 一个例题
- 解题方法
- 常见错误
- 关联关卡
- 掌握状态

### MVP 知识卡

| 知识卡     | 对应知识点  |
| ------- | ------ |
| 平均分线索卡  | 平均分    |
| 除法算式线索卡 | 除法算式含义 |
| 剩余徽章线索卡 | 有余数除法  |
| 余数规则线索卡 | 余数小于除数 |

### 验收标准

- 用户通过普通关卡后解锁对应知识卡。
- 用户可以从地图进入线索库。
- 线索库展示已解锁知识卡。
- 未解锁知识卡可以显示为灰色或隐藏。
- 知识卡内容适合二年级孩子阅读。

---

## 8.10 勋章馆

### 功能说明

记录孩子在学习过程中的成就。

### MVP 勋章设计

| 勋章    | 获得条件        | 是否重复获得 |
| ----- | ----------- | ------ |
| 新手小侦探 | 完成第一个普通关卡   | 否      |
| 线索收集员 | 收集全部 4 条线索  | 否      |
| 悬案调查员 | 第一次进入悬案馆复习  | 否      |
| 坚持不放弃 | 错题重做并答对     | 否      |
| 破案小能手 | 完成 Boss 并结案 | 否      |

### 勋章记录内容

- 勋章名称
- 勋章图标
- 获得时间
- 获得原因
- 对应案件

### 验收标准

- 达成条件后自动发放勋章。
- 勋章馆能展示已获得勋章。
- 未获得勋章可以显示为灰色。
- 勋章记录包含获得时间和获得原因。
- 同一枚勋章不会重复发放。

## 9. 题目系统需求

### 9.1 题型范围

MVP 支持以下题型：

| 题型     | 是否支持 |
| ------ | ---- |
| 单选题    | 支持   |
| 填空题    | 支持   |
| 判断题    | 支持   |

| 拖拽题    | 暂不支持 |
| 手写题    | 暂不支持 |
| 多步骤解答题 | 暂不支持 |

### 9.2 题目字段

| 字段              | 说明        |

| semester        | 学期        |
| unit            | 单元        |
| knowledgePoint  | 知识点       |
| levelId         | 所属关卡      |
| questionType    | 题型        |
| difficulty      | 难度        |
| stem            | 题干        |
| options         | 选项        |
| answer          | 正确答案      |
| explanation     | 解析        |
| wrongHint       | 答错提示      |
| isBossQuestion  | 是否 Boss 题 |
| wrongCategory   | 错题分类      |

### 9.3 题库数量建议

| 类型       | 数量   |
| -------- | ---- |
| 普通关卡正式题  | 16 道 |
| Boss 正式题 | 2 道  |
| 备用题      | 10 道 |
| 合计       | 28 道 |

### 9.4 题目文案原则

- 题干尽量短。
- 使用二年级孩子能理解的词语。
- 数字不要过大。
- 尽量结合案件情境。
- 错误提示避免否定和打击。
- 解析用“一步一步说清楚”的方式表达。

## 13. MVP 不做清单

为避免范围失控，MVP 明确不做以下内容：

| 功能      | 不做原因                |
| ------- | ------------------- |
| 家长端     | 会增加账号、数据报表和管理复杂度    |
| 老师端     | 涉及班级、作业、学生管理        |
| 排行榜     | 与首期学习闭环无关，且可能增加比较压力 |
| 商城系统    | 容易偏离学习目标            |
| 宠物养成    | 玩法成本较高，暂不验证         |
| 多人协作    | 技术和规则复杂             |
| AI 自动出题 | 题目质量难以控制            |
| 语音识别    | 成本高，非核心闭环           |
| 大量动画剧情  | 美术和开发成本高            |
| 全册内容    | 内容量大，不适合 MVP        |
| 多学科     | 会稀释验证重点             |

## 14. 快速验证计划

### 14.1 验证目标

MVP 完成后，需要验证：

1. 孩子是否愿意完成一个完整案件。
2. 孩子是否能理解“接案、闯关、收集线索、挑战 Boss”的流程。
3. 探案包装是否比普通刷题更有吸引力。
4. 错题进入悬案馆后，孩子是否愿意回来复习。
5. 线索库是否能帮助孩子理解知识点。
6. 勋章是否能带来成就感。
7. 家长是否认可这种学习形式。

### 14.2 测试对象

```
5～10 名小学二年级下册或接近该水平的孩子。
```

### 14.3 测试方式

- 家庭或现场测试均可。
- 每个孩子体验 10～15 分钟。
- 尽量观察孩子独立操作过程。
- 家长可以旁观，但尽量不直接提示。
- 测试后分别收集孩子反馈和家长反馈。

### 14.4 成功指标

| 指标      | 成功标准                 |
| ------- | -------------------- |
| 完成率     | 70% 以上孩子能完成首个案件      |
| 继续意愿    | 60% 以上孩子表示愿意继续玩下一案   |
| 流程理解    | 70% 以上孩子知道下一步该做什么    |
| Boss 难度 | Boss 通过率在 50%～80% 之间 |
| 悬案馆吸引力  | 50% 以上孩子愿意查看或重做错题    |
| 勋章吸引力   | 50% 以上孩子愿意查看勋章       |
| 家长认可度   | 70% 以上家长认为形式有学习价值    |

### 14.5 观察记录问题

测试时建议记录：

```
孩子是否能独立进入案件？
孩子是否能理解剧情？
孩子在哪一步停顿或困惑？
孩子答错后是否有挫败感？
孩子是否愿意继续挑战？
孩子是否主动查看线索库或悬案馆？
孩子获得勋章时是否有明显正反馈？
家长是否认为题目难度合适？
家长是否愿意让孩子再次使用？
```

## 16. 风险与控制

### 16.1 范围失控风险

风险：

```
开发过程中不断增加地图、角色、动画、排行榜、商城等功能，导致 MVP 无法完成。
```

控制方式：

```
严格执行 MVP 不做清单。
所有新增想法放入后续版本池。
```

### 16.2 教育内容质量风险

风险：

```
题目有趣但知识点不准确，或知识点准确但游戏体验无聊。
```

控制方式：

```
题目必须绑定知识点。
每道题必须有解析。
Boss 必须考察综合应用，而不是单纯变难。
```

### 16.3 儿童阅读难度风险

风险：

```
剧情和题干太长，二年级孩子读不懂。
```

控制方式：

```
文案短句化。
每屏只表达一个核心信息。
后续可考虑增加语音朗读。
```

### 16.4 游戏喧宾夺主风险

风险：

```
孩子只关注奖励，不关注数学学习。
```

控制方式：

```
奖励必须绑定学习行为。
不做商城和复杂养成。
线索、勋章、结案都与知识点和答题表现关联。
```

### 16.5 IP 风险

风险：

```
如果直接使用疯狂动物城相关角色和地图，可能涉及版权问题。
```

控制方式：

```
MVP 使用原创动物城市设定。
不使用已有 IP 名称、角色、地图和形象。
```

## 18. 项目总原则

```
MVP 的目标不是做出完整的动物学习大世界，而是做出一个孩子愿意完成、家长认可有效、团队能够继续扩展的探案式数学学习闭环。
```

在 MVP 阶段，所有功能都必须服务于这条主线：

```
学习除法知识
→ 完成探案闯关
→ 记录错题
→ 复习知识
→ 获得正反馈
```
