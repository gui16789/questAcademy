## 2. 文档目的

本文档定义 v0.2.0 内容包校验脚本 `scripts/validate-content.mjs` 的校验范围、错误等级、输出格式和发布门槛。

v0.2.0 暂不引入 Ajv 和完整 JSON Schema，先使用 Node.js 原生脚本完成必填字段和引用完整性检查。

## 3. 校验目标

内容校验要保证：

- 内容包结构完整。
- ID 唯一。
- 必填字段存在。
- 引用关系有效。
- 题目绑定教材、知识点、能力点和易错点。
- Boss 独立配置且步骤题存在。
- 复习规则能找到对应知识卡和题型。
- 发布前不存在阻塞性内容错误。

## 4. 命令

```bash
node scripts/validate-content.mjs
```

默认校验：

```text
content/math/bsd/grade-2/semester-2/unit-1-division
```

后续可扩展为：

```bash
node scripts/validate-content.mjs content/math/bsd/grade-2/semester-2/unit-1-division
```

## 5. 输出格式

### 5.1 通过

```text
Content validation passed.
Checked package: math.bsd.g2.s2.unit-1-division@0.1.0
Errors: 0
Warnings: 0
```

### 5.2 失败

```text
Content validation failed.
ERROR [questions/level-remainder-rule.json:q14] missing skillId
ERROR [cases/case-carrot-badge.json] unknown levelId level-x
Warnings: 1
```

存在 `ERROR` 时退出码必须非 0。

## 6. 错误等级

| 等级 | 含义 | 是否阻塞发布 |
| --- | --- | --- |
| ERROR | 内容无法可靠运行或影响学习准确性 | 是 |
| WARNING | 不阻塞运行，但需要关注 | 否 |

## 7. 必校验文件

内容包必须包含：

```text
manifest.json
textbook.json
knowledge-map.json
cases/case-carrot-badge.json
questions/level-average-sharing.json
questions/level-division-expression.json
questions/level-with-remainder.json
questions/level-remainder-rule.json
questions/boss-carrot-badge-final.json
questions/reserve.json
cards/knowledge-cards.json
clues.json
badges.json
review-rules.json
```

缺失任一文件为 `ERROR`。

## 8. manifest 校验

必填字段：

```text
schemaVersion
contentPackageId
contentVersion
status
subjectId
textbookVersionId
gradeId
semesterId
unitId
title
defaultCaseId
locale
```

校验规则：

- `contentPackageId` 必须等于 `math.bsd.g2.s2.unit-1-division`。
- `defaultCaseId` 必须能在 cases 中找到。
- `contentVersion` 必须符合 `MAJOR.MINOR.PATCH`。

## 9. textbook 校验

必须包含：

- subject。
- textbookVersion。
- grade。
- semester。
- unit。
- curriculumNodes。

教材节点规则：

- 每个 curriculumNode ID 唯一。
- 每个 curriculumNode 必须映射至少一个 knowledgePointId。
- 映射的 knowledgePointId 必须存在于 knowledge-map。

## 10. knowledge-map 校验

必须包含：

- knowledgePoints。
- skills。
- misconceptions。
- questionPatterns。

校验规则：

- 每个 knowledgePointId 唯一。
- 每个 skillId 唯一。
- 每个 misconceptionId 唯一。
- 每个 questionPatternId 唯一。
- skill 引用的 knowledgePointId 必须存在。
- misconception 引用的 knowledgePointId 和 skillId 必须存在。
- questionPattern 引用的 knowledgePointId 和 skillId 必须存在。

## 11. case / level 校验

Case 必填：

```text
caseId
contentPackageId
name
unitId
targetKnowledgePointIds
levelIds
bossTaskId
summary
```

Level 必填：

```text
levelId
caseId
order
name
knowledgePointId
skillIds
questionGroupId
passRule
reward
unlockRule
```

校验规则：

- case 的 `levelIds` 必须能找到对应 level。
- level 的 knowledgePointId 必须存在。
- level 的 skillIds 必须存在。
- reward 中的 clueId 和 knowledgeCardIds 必须存在。

## 12. question 校验

每道题必填：

```text
questionId
contentVersion
subjectId
textbookVersionId
gradeId
semesterId
unitId
lessonId
curriculumNodeId
knowledgePointId
skillId
misconceptionId
questionPatternId
caseId
levelId
questionType
difficultyBand
stem
answer
explanation
wrongHint
status
```

校验规则：

- questionId 全包唯一。
- knowledgePointId 必须存在。
- skillId 必须存在。
- misconceptionId 必须存在。
- questionPatternId 必须存在。
- curriculumNodeId 必须存在。
- levelId 必须存在，备用题除外。
- questionType 必须属于允许枚举。
- single / judge 类型必须有 options。
- answer 不能为空。
- explanation 不能为空。

允许 questionType：

```text
single
judge
fill
```

## 13. knowledge-card 校验

必填字段：

```text
knowledgeCardId
contentVersion
knowledgePointId
skillIds
title
summary
example
method
commonMistake
reviewPrompt
status
```

校验规则：

- knowledgeCardId 唯一。
- knowledgePointId 必须存在。
- skillIds 必须存在。
- summary 不建议超过 40 个中文字符，超过为 WARNING。

## 14. clue 校验

必填字段：

```text
clueId
caseId
levelId
knowledgePointId
title
text
unlockCondition
```

校验规则：

- clueId 唯一。
- caseId 必须存在。
- levelId 必须存在。
- knowledgePointId 必须存在。
- text 不能为空。

## 15. boss 校验

BossTask 必填：

```text
bossTaskId
caseId
name
targetKnowledgePointIds
targetSkillIds
scenario
steps
passRule
failureReviewRoute
```

校验规则：

- bossTaskId 必须与 case.bossTaskId 对应。
- targetKnowledgePointIds 必须存在。
- targetSkillIds 必须存在。
- steps 至少 1 个。
- 每个 step 的 questionId 必须存在。
- passRule 必须存在。
- failureReviewRoute 中的 knowledgeCardId 必须存在。

## 16. badge 校验

必填字段：

```text
badgeId
name
description
icon
triggerRule
isRepeatable
```

校验规则：

- badgeId 唯一。
- triggerRule 引用的 caseId 或 levelId 必须存在。
- isRepeatable 必须是布尔值。

## 17. review-rule 校验

必填字段：

```text
reviewRuleId
knowledgePointId
skillId
misconceptionId
trigger
actions
```

校验规则：

- reviewRuleId 唯一。
- knowledgePointId 必须存在。
- skillId 必须存在。
- misconceptionId 必须存在。
- actions 中引用的 knowledgeCardId 必须存在。
- actions 中引用的 questionPatternId 必须存在。

## 18. 发布门槛

v0.2.0 发布前必须满足：

```text
validate-content 无 ERROR
题目答案和解析已审核
Boss 引用完整
复习规则引用完整
内容包版本确认
```

WARNING 可以不阻塞发布，但必须记录在 Release Notes 或内容审核记录中。
