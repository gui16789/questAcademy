## 下一阶段优先推进

v1.0.0 已完成发布，下一阶段建议按以下顺序推进：

1. 完成 v1.1.0 发布前检查清单。
2. 合并 v1.1.0 发布准备 PR。
3. 确认无 P0 / P1 问题。
4. 发布前由项目负责人确认是否创建 `v1.1.0` tag 和 GitHub Release。

## 文档原则

- 重要决策必须进入文档。
- 当前版本的范围、验收标准和不做清单需要冻结。
- 内容、技术和设计变更要同步更新对应文档。
- 版本结束后需要记录复盘和后续改进项。
