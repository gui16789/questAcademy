## 3. Issue 1：v1.2.0 规划和范围冻结

目标：

- 创建 v1.2.0 规划文档。
- 更新文档中心。
- 明确本版本包含、不包含和验收标准。

不做：

- 不改运行时代码。
- 不新增内容包。

验收：

- 规划文档已合并。
- 文档中心能找到 v1.2.0 入口。

## 4. Issue 2：轻量课程地图 / 单元地点入口

目标：

- 顶部或开始页提供轻量课程地图 / 单元地点入口。
- 入口读取 registry，不硬编码课程或地点列表。
- 产品概念为：语文、数学、英语是课程；教材单元是课程地图上的地点节点；内容包是某个地点的运行包。
- 选择内容包后通过 `contentPackageId` 加载对应内容。
- 启动页课程、地点和案件名从 registry / 内容包派生。

不做：

- 不做复杂课程中心。
- 不做推荐、筛选、购买、账号绑定。

验收：

- 默认内容包体验不变。
- 移动端无横向溢出。
- 当前只有一个内容包时，入口仍稳定可用。

## 5. Issue 3：第二内容包选题和审核记录

目标：

- 确定第二内容包的教材、年级、学期、单元和内容范围。
- 新建 `docs/reviews/{contentPackageId}@{contentVersion}-review.md`。
- 按内容工厂交付清单记录审核状态。

不做：

- 不直接把未审核内容包标记为 `published`。

验收：

- 审核记录包含内容审核、教研审核、儿童可读性审核和技术检查。
- P0 / P1 阻塞项明确记录。

## 6. Issue 4：第二内容包 draft 资产接入

目标：

- 新增第二内容包目录和 JSON 内容。
- 完成 manifest 文件引用。
- registry 新增第二内容包 entry，状态保持 `draft` 或对应审核状态。

不做：

- 不批量新增多个内容包。
- 不改变现有除法内容包版本。

验收：

- `npm run validate:content -- --package {contentPackageId}` 通过。
- 默认内容包仍可校验通过。

## 7. Issue 5：多内容包 loader import map 扩展

目标：

- `apps/web` 可以加载 registry 中新增的 bundled 内容包。
- loader 错误信息清楚，缺少 bundled 文件时明确失败。

验收：

- `?contentPackageId={contentPackageId}` 可打开第二内容包。
- 地点入口切换后进入对应内容包。
- 不同内容包 localStorage key 隔离。

## 8. Issue 6：多内容包校验与回归补强

目标：

- `validate-content --all` 覆盖默认包和第二内容包。
- `test:regression --all` 覆盖默认包和第二内容包。
- 回归记录纳入发布检查。

验收：

- 所有 registry 内容包校验通过。
- 所有 registry 内容包浏览器回归通过。
- 320px 移动端无横向溢出。

## 9. Issue 7：v1.2.0 发布准备

目标：

- 更新 README、Release Notes 和发布前检查清单。
- 确认产品版本、内容包版本和内容包状态。
- 跑完整发布命令。

验收：

- 无 P0 / P1 已知问题。
- 发布停点前等待项目负责人确认。
