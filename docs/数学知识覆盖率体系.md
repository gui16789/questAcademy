# 数学知识覆盖率体系

本文档说明数学内容包的覆盖率体系、脚本指标和后续补题流程。当前第一版覆盖北师大版数学二年级下册 8 个单元，不改变学生端学习主流程，不新增后端。

## 数据入口

- Coverage plan: `content/coverage-plans/math/bsd/grade-2/semester-2.json`
- 内容包声明: 每个 `manifest.json` 的 `coverage` 字段
- 统计脚本: `npm run report:coverage -- --all`
- JSON 输出: `npm run --silent report:coverage -- --all --json`
- 内容校验: `npm run validate:content -- --all`

## 核心概念

- `expectedKnowledgePoint`: 内容包应覆盖的知识点，来自全册 coverage plan，而不是只看内容包当前已有题目。
- `microKnowledgePoint`: 诊断粒度的微知识点。当前版本用现有 `skill` 作为第一版微知识点粒度，后续可以在题目上补充 `microKnowledgePointId` 做更细拆分。
- `skill`: 能力点，必须有普通诊断题支撑。
- `misconception`: 常见易错点，至少要有 1 道针对性普通诊断题。
- `coverageTarget`: 覆盖率目标。当前数学二下要求知识点、微知识点、skill、misconception 都达到 100% 才可视为发布就绪。
- `diagnosticRequirement`: 诊断题最低要求。当前要求每个 required 微知识点至少 2 道普通诊断题、每个 skill 至少 2 道普通诊断题、每个 misconception 至少 1 道普通诊断题。
- `coverageStatus`: 当前覆盖状态。内容包 draft 阶段允许 `draft_gap_known` 并输出 warning；published 阶段同样缺口会升级为 error。

## 统计口径

- 普通诊断覆盖只统计 `levelId` 不是 `boss`、也不是 `reserve` 的题目。
- `reserve` 题永远不计入正式覆盖率。
- Boss 题单独统计，可以用于综合体验和结案挑战，但不能替代基础知识点、skill 或 misconception 覆盖。
- 体验关卡覆盖统计关卡和 Boss 触达情况，用来观察小朋友是否在流程里见过该内容。
- 诊断题库覆盖统计普通题数量，用来判断是否具备可诊断性。

## 后续补题流程

1. 先运行 `npm run --silent report:coverage -- --all --json`，从 `uncoveredMicroKnowledgePoints`、`lowCoverageMicroKnowledgePoints`、`weakDiagnosticMicroKnowledgePoints` 取补题清单。
2. 每次补题优先补普通关卡题库，不用 Boss 题抵扣基础覆盖。
3. 新题必须绑定正确的 `knowledgePointId`、`skillId`、`misconceptionId`、`questionPatternId`；需要更细粒度时再补 `microKnowledgePointId`。
4. 补题后运行 `npm run validate:content -- --all` 和 `npm run report:coverage -- --all`，确认 draft warning 数下降。
5. 内容包进入 `published` 前，覆盖率 warning 必须清零，否则校验会升级为 error。
