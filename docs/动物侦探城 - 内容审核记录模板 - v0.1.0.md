## 2. 文档目的

本文档用于记录内容包发布前的内容审核、教研审核、儿童可读性审核和技术 Schema 检查。

目标是保证：

- 内容字段完整。
- 教材和知识点准确。
- 题目答案和解析正确。
- 文案适合低年级儿童。
- 内容包结构和引用关系正确。
- 发布前有清晰审核留痕。

## 3. 已确认关键决策

| 编号 | 决策 | 说明 |
| --- | --- | --- |
| D1 | 审核分四类 | 内容审核、教研审核、儿童可读性审核、技术 Schema 检查 |
| D2 | 审核对象覆盖完整内容包 | 教材节点、知识图谱、题目、解析、知识卡、线索、Boss、勋章、复习规则、manifest |
| D3 | 审核状态固定 | pending、approved、changes_requested、blocked |
| D4 | 记录分两层 | 内容包级审核总表 + 单项问题记录表 |
| D5 | 全部 approved 才能 published | 任一 blocked 或 changes_requested 都不能发布 |

## 4. 审核状态定义

| 状态 | 含义 |
| --- | --- |
| pending | 待审核 |
| approved | 通过 |
| changes_requested | 需要修改后复审 |
| blocked | 阻塞发布 |
