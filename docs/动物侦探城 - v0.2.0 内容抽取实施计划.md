## 2. 文档目的

本文档定义 v0.2.0 的内容抽取实施范围、目录结构、Issue / PR 拆分、内容加载方式、数据兼容策略、内容校验脚本和验收标准。

v0.2.0 是从静态 MVP 走向内容包架构的第一步。

## 3. 已确认关键决策

| 编号 | 决策 | 说明 |
| --- | --- | --- |
| D1 | v0.2.0 只做结构化改造 | 不新增玩法、不扩新案件、不接后端 |
| D2 | 保留当前静态架构 | 继续使用 `index.html / styles.css / app.js` |
| D3 | 新增 `content/` 和 `scripts/` | 内容包和校验脚本落地 |
| D4 | 内容包按徽章失踪案样例拆分 | manifest、textbook、knowledge-map、cases、questions、cards 等 |
| D5 | 实施拆成 6 个 Issue / PR | 降低风险，便于 Review |
| D6 | 推荐本地静态服务运行 | 不保证 `file://` 下 fetch JSON 可用 |
| D7 | v0.2.0 增加数据版本并兼容 v0.1.0 | 避免本地进度直接丢失 |
| D8 | 校验脚本先用 Node.js 原生脚本 | v1.0 再考虑 JSON Schema + Ajv |

## 7. 运行方式

### 7.1 推荐方式

使用本地静态服务：

```bash
python -m http.server 4173 --bind 127.0.0.1
```

访问：

```text
http://127.0.0.1:4173
```

### 7.2 不保证方式

不保证直接 `file://` 打开后能正常 `fetch` JSON。

原因：

```text
浏览器可能限制本地文件跨文件读取。
```

README 需要同步更新运行说明。

## 8. Issue / PR 拆分计划

所有实施必须按：

```text
Issue -> 独立分支 -> PR -> Review -> 合并 main
```

### 8.1 Issue 1：建立内容包基础

目标：

```text
建立 content 目录和 manifest/textbook/knowledge-map。
```

文件：

```text
content/math/bsd/grade-2/semester-2/unit-1-division/manifest.json
content/math/bsd/grade-2/semester-2/unit-1-division/textbook.json
content/math/bsd/grade-2/semester-2/unit-1-division/knowledge-map.json
```

验收：

- manifest 字段完整。
- 教材节点完整。
- 知识点、能力点、易错点结构完整。
- ID 与文档一致。

### 8.2 Issue 2：抽取案件、关卡、线索、知识卡、勋章

目标：

```text
把 cases/levels/clues/cards/badges 从 app.js 抽成 JSON。
```

文件：

```text
cases/case-carrot-badge.json
cards/knowledge-cards.json
clues.json
badges.json
```

验收：

- 案件包含 4 个关卡。
- 每个关卡引用知识点。
- 线索引用关卡和知识点。
- 知识卡引用知识点。
- 勋章触发规则明确。

### 8.3 Issue 3：抽取题目和 Boss

目标：

```text
把 16 道正式题、10 道备用题、2 道 Boss 题抽成 JSON。
```

文件：

```text
questions/level-average-sharing.json
questions/level-division-expression.json
questions/level-with-remainder.json
questions/level-remainder-rule.json
questions/boss-carrot-badge-final.json
questions/reserve.json
```

验收：

- 每道题有教材字段。
- 每道题有 knowledgePointId。
- 每道题有 skillId。
- 每道题有 misconceptionId。
- 每道题有 questionPatternId。
- Boss 独立配置。

### 8.4 Issue 4：增加内容加载器

目标：

```text
让 app.js 从 JSON 内容包读取内容。
```

建议实现：

```text
loadContentPackage()
loadJson(path)
buildRuntimeContent()
```

验收：

- 页面体验基本不变。
- 普通关卡能读取 JSON 题目。
- 知识卡、线索、勋章来自 JSON。
- Boss 来自 JSON。
- 内容加载失败时有友好提示。

### 8.5 Issue 5：增加内容校验脚本

目标：

```text
增加 scripts/validate-content.mjs。
```

命令：

```bash
node scripts/validate-content.mjs
```

校验：

- ID 唯一。
- 必填字段存在。
- 关卡引用题目存在。
- 题目引用知识点、能力点、易错点存在。
- 知识卡引用知识点存在。
- 线索引用关卡存在。
- Boss 步骤题存在。
- 复习规则引用完整。

验收：

- 校验成功时退出码为 0。
- 校验失败时退出码非 0。
- 错误信息能定位文件和 ID。

### 8.6 Issue 6：数据版本与回归测试

目标：

```text
补齐 dataVersion / contentPackageId / contentVersion，并回归核心流程。
```

要求：

- 新数据写入 `dataVersion: "0.2.0"`。
- 新数据写入 `contentPackageId`。
- 新数据写入 `contentVersion`。
- 兼容读取 v0.1.0 本地记录。
- 不直接清空用户进度。

验收：

- v0.1.0 记录可读取或安全迁移。
- 新记录有版本字段。
- 刷新后进度不丢失。
- 核心流程完整跑通。

## 9. 内容加载设计

### 9.1 加载入口

建议在 `app.js` 中新增：

```js
const CONTENT_PACKAGE_PATH = "content/math/bsd/grade-2/semester-2/unit-1-division";
```

加载：

```js
async function loadContentPackage(basePath) {
  // load manifest, textbook, knowledge-map, case, questions, cards, clues, badges, review-rules
}
```

### 9.2 启动流程变化

当前：

```text
定义静态数组
-> loadState
-> render
```

v0.2.0：

```text
loadContentPackage
-> buildRuntimeContent
-> loadState
-> render
```

### 9.3 加载失败

加载失败时显示：

```text
内容包没有加载成功，请使用本地静态服务打开。
```

并提示：

```text
python -m http.server 4173 --bind 127.0.0.1
```

## 10. 数据兼容策略

### 10.1 v0.1.0 旧数据

旧数据可能没有：

```text
dataVersion
contentPackageId
contentVersion
knowledgePointId
skillId
misconceptionId
```

### 10.2 v0.2.0 新数据

新数据必须包含：

```json
{
  "dataVersion": "0.2.0",
  "contentPackageId": "math.bsd.g2.s2.unit-1-division",
  "contentVersion": "0.1.0"
}
```

### 10.3 迁移策略

读取旧数据时：

```text
如果缺少 dataVersion，视为 v0.1.0。
补默认 contentPackageId。
补默认 contentVersion。
保留已有进度、错题和勋章。
无法映射的旧错题保留题目摘要，并标记 question_migrated=false。
```

## 13. 风险与控制

| 风险 | 控制 |
| --- | --- |
| fetch JSON 导致 file:// 不可用 | README 明确使用静态服务 |
| 内容抽取时字段漏掉 | validate-content 校验 |
| ID 改名导致进度丢失 | 保持旧 ID 或提供映射 |
| app.js 改动过大 | 分 6 个 PR 实施 |
| 题目内容迁移出错 | 内容包样例和测试清单对照 |
| v0.2.0 趁机加玩法 | 范围冻结，只做结构化 |

## 14. v0.2.0 验收标准

必须满足：

- 内容已抽成 JSON。
- 现有 MVP 体验基本不变。
- 内容校验脚本通过。
- 本地学习记录有版本字段。
- v0.1.0 数据可兼容读取或安全迁移。
- 核心学习闭环完整跑通。
- 无 P0 / P1 Bug。
- README 已更新运行方式。
