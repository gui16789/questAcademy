## 2. 文档目的

本文档定义《动物侦探城》的埋点事件、指标口径、数据维度、隐私边界和阶段判断指标。

目标不是一开始接入复杂分析系统，而是先统一：

- 什么行为值得记录。
- 每个事件带什么字段。
- 如何判断学习闭环是否成立。
- 如何通过数据改进题目、知识卡、Boss 和流程。

## 3. 已确认关键决策

| 编号 | 决策 | 说明 |
| --- | --- | --- |
| D1 | v0.1.0 不接第三方分析 SDK | 先定义事件规范，用本地日志或手工记录 |
| D2 | 指标分 5 类 | 流程、学习、复习、体验、内容质量 |
| D3 | 定义核心事件 | 覆盖开始、关卡、题目、错题、Boss、结案、勋章 |
| D4 | 学习事件必须带知识维度 | 支持知识点、能力点、错因分析 |
| D5 | v0.x 遵守儿童隐私最小化 | 不采集真实姓名、手机号、定位和设备唯一标识 |
| D6 | 阶段门使用完成率、Boss 通过率、复习率、继续意愿和家长认可度 | 支持是否进入下一阶段判断 |

## 4. 埋点实现阶段

### 4.1 v0.1.0

```text
定义事件规范
不强制实现线上采集
通过手工测试记录和观察表收集核心指标
```

### 4.2 v0.2.0

```text
可在本地维护 events 数组
事件写入 localStorage 或导出测试记录
仍不上传个人学习数据
```

### 4.3 v1.0.0

```text
根据需要评估真实分析服务
上线前补充隐私说明和数据采集边界
```

## 5. 指标分层

| 指标类型 | 关注问题 | 示例 |
| --- | --- | --- |
| 流程指标 | 孩子能否完成主流程 | 开始率、接案率、关卡完成率、结案率 |
| 学习指标 | 孩子是否掌握知识 | 正确率、错题率、Boss 通过率、知识点掌握 |
| 复习指标 | 错题和知识卡是否有用 | 悬案馆进入率、错题侦破率、线索库查看率 |
| 体验指标 | 孩子是否愿意继续 | 停顿点、退出点、勋章查看、继续意愿 |
| 内容质量指标 | 内容是否准确和适龄 | 题目错误率、错题集中度、教研返工率 |

## 6. 公共事件字段

所有事件建议包含：

```json
{
  "eventName": "question_answered",
  "eventId": "uuid",
  "anonymousUserId": "anon-001",
  "sessionId": "session-001",
  "timestamp": "2026-05-22T00:00:00.000Z",
  "appVersion": "0.1.0",
  "dataVersion": "0.1.0",
  "source": "local_test"
}
```

v0.x 阶段使用匿名用户 ID，不采集真实身份。

## 7. 学习维度字段

凡是学习相关事件，应尽量包含：

```json
{
  "contentPackageId": "math.bsd.g2.s2.unit-1-division",
  "contentVersion": "0.1.0",
  "subjectId": "math",
  "textbookVersionId": "bsd",
  "gradeId": "grade-2",
  "semesterId": "semester-2",
  "unitId": "unit-1-division",
  "caseId": "case-carrot-badge",
  "levelId": "level-remainder-rule",
  "questionId": "q14",
  "knowledgePointId": "math.division.remainder-rule",
  "skillId": "skill.remainder.compare-with-divisor",
  "misconceptionId": "mis.remainder.greater-than-divisor"
}
```

这样后续可以按教材、单元、知识点、能力点和错因分析。

## 8. 核心事件列表

| 事件名 | 触发时机 | 主要用途 |
| --- | --- | --- |
| app_start | 打开产品 | 统计访问和会话开始 |
| nickname_submitted | 提交昵称 | 统计开始意愿 |
| map_viewed | 进入地图 | 统计主入口到达 |
| case_opened | 打开案件 | 统计接案率 |
| level_started | 开始关卡 | 统计关卡开始 |
| question_answered | 提交答案 | 学习分析核心事件 |
| level_completed | 完成关卡 | 统计通过率 |
| clue_unlocked | 获得线索 | 统计线索进度 |
| knowledge_card_viewed | 查看知识卡 | 统计线索库使用 |
| wrong_added | 错题进入悬案馆 | 统计错题产生 |
| wrong_review_started | 开始重做错题 | 统计错题复习意愿 |
| wrong_solved | 错题侦破 | 统计错题复习效果 |
| boss_unlocked | Boss 解锁 | 统计流程进度 |
| boss_started | 开始 Boss | 统计挑战意愿 |
| boss_completed | 完成 Boss | 统计 Boss 通过率 |
| case_closed | 结案 | 统计案件完成 |
| badge_earned | 获得勋章 | 统计激励触发 |
| badge_hall_viewed | 查看勋章馆 | 统计成就吸引力 |
| session_ended | 体验结束 | 统计会话长度和退出点 |

## 9. 事件定义

### 9.1 app_start

触发：

```text
产品加载完成。
```

字段：

```json
{
  "eventName": "app_start",
  "entryType": "fresh_open",
  "hasExistingProgress": true
}
```

### 9.2 case_opened

触发：

```text
用户进入案件详情页。
```

字段：

```json
{
  "eventName": "case_opened",
  "caseId": "case-carrot-badge",
  "caseStatus": "not_started"
}
```

### 9.3 level_started

触发：

```text
用户进入普通关卡。
```

字段：

```json
{
  "eventName": "level_started",
  "caseId": "case-carrot-badge",
  "levelId": "level-average-sharing",
  "knowledgePointId": "math.division.average-sharing",
  "attemptIndex": 1
}
```

### 9.4 question_answered

触发：

```text
用户提交题目答案。
```

字段：

```json
{
  "eventName": "question_answered",
  "questionId": "q14",
  "questionType": "judge",
  "difficultyBand": "basic",
  "knowledgePointId": "math.division.remainder-rule",
  "skillId": "skill.remainder.compare-with-divisor",
  "misconceptionId": "mis.remainder.greater-than-divisor",
  "isCorrect": false,
  "attemptIndex": 1,
  "timeSpentSeconds": 18,
  "isBossQuestion": false
}
```

用途：

- 计算题目正确率。
- 计算知识点正确率。
- 识别高频易错点。
- 判断题目难度是否合理。

## 10. 核心指标口径

### 10.1 流程指标

| 指标 | 计算方式 |
| --- | --- |
| 案件开始率 | case_opened 用户数 / app_start 用户数 |
| 关卡开始率 | level_started 用户数 / case_opened 用户数 |
| 关卡完成率 | level_completed 用户数 / level_started 用户数 |
| Boss 开始率 | boss_started 用户数 / boss_unlocked 用户数 |
| 案件完成率 | case_closed 用户数 / case_opened 用户数 |

### 10.2 学习指标

| 指标 | 计算方式 |
| --- | --- |
| 题目正确率 | 正确 answer 数 / 总 answer 数 |
| 知识点正确率 | 某知识点正确 answer 数 / 某知识点总 answer 数 |
| 错题产生率 | wrong_added 数 / question_answered 数 |
| Boss 通过率 | boss_completed 且 isPassed=true / boss_completed |
| 能力点错误率 | 某 skill 错误数 / 某 skill 答题数 |

### 10.3 复习指标

| 指标 | 计算方式 |
| --- | --- |
| 悬案馆进入率 | wrong_review_started 用户数 / 有 wrong_added 用户数 |
| 错题侦破率 | wrong_solved 数 / wrong_added 数 |
| 线索库进入率 | knowledge_card_viewed 用户数 / clue_unlocked 用户数 |
| 知识卡查看率 | knowledge_card_viewed 次数 / 已解锁知识卡数 |

### 10.4 体验指标

| 指标 | 计算方式 |
| --- | --- |
| 勋章查看率 | badge_hall_viewed 用户数 / badge_earned 用户数 |
| 继续体验意愿 | 测试后表示愿意继续的孩子数 / 测试孩子数 |
| 流程理解率 | 能说出下一步或独立操作的孩子数 / 测试孩子数 |
| 停顿点 | 观察记录中停顿超过预设时间的页面或操作 |

### 10.5 内容质量指标

| 指标 | 计算方式 |
| --- | --- |
| 高频错题 | 单题错误率明显高于同关卡平均 |
| 错题集中度 | 某知识点或能力点错误占全部错误比例 |
| 教研返工率 | 审核退回内容数 / 提交审核内容数 |
| Boss 难度合理性 | Boss 通过率是否在 50% - 80% |

## 11. MVP 阶段门指标

### 11.1 进入 v0.2.0

v0.2.0 是结构化改造，不要求所有儿童指标达标。

建议满足：

```text
核心流程可跑通
关键事件定义完整
能通过手工或本地记录观察主要指标
无 P0 / P1 Bug
```

### 11.2 进入 v1.0

建议满足：

| 指标 | 目标 |
| --- | --- |
| 案件完成率 | >= 70% |
| 流程理解率 | >= 70% |
| Boss 通过率 | 50% - 80% |
| 悬案馆进入率 | >= 50% |
| 错题侦破率 | >= 40% |
| 线索库进入率 | >= 50% |
| 继续体验意愿 | >= 60% |
| 家长认可度 | >= 70% |

## 12. 儿童隐私边界

v0.x 阶段：

```text
不采集真实姓名
不采集手机号
不采集定位
不采集设备唯一标识
不上传个人学习数据
只在本地或测试记录中使用匿名编号
```

儿童测试记录中使用：

```text
child-001
child-002
```

不使用真实姓名。
