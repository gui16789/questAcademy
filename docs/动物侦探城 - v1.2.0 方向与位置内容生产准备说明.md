## 4. 推荐知识点拆解草案

以下拆解已用于 AI draft 的 `knowledge-map.json`，但仍只是内容生产讨论起点，必须由教研负责人确认后才能推进到 `approved` 或 `published`。

| 知识点 ID 草案 | 名称 | 学习目标草案 | 风险 |
| --- | --- | --- | --- |
| `math.direction.cardinal` | 东、南、西、北 | 能辨认四个基本方向 | 需要明确“上北下南左西右东”的平面图约定 |
| `math.direction.map-position` | 平面图中的位置 | 能在简单平面图中描述物体位置 | 图形表达要足够简单 |
| `math.direction.diagonal` | 东北、东南、西北、西南 | 能辨认四个斜方向 | 斜方向组合词容易读反 |
| `math.direction.route` | 简单路线描述 | 能按方向词描述简单路线 | 可能需要路线图或排序交互 |

## 7. v1.2.0 建议范围

建议 v1.2.0 首批方向与位置 draft 内容包优先使用：

- `single`
- `judge`

暂不阻塞 v1.2.0 的题型：

- `sort`
- `drag`
- 需要地图点击的自定义题型

原因：

- 当前 React 学生端已稳定支持 `single`、`judge`、`fill`。
- `fill` 当前是数字键盘，方向词填空会造成输入体验不匹配。
- `sort`、`drag` 虽然在 schema 中有类型，但 UI 和回归脚本尚未覆盖。

## 8. 案件包装草案

候选方向：

```text
动物侦探城方向邮局 / 迷路包裹案
```

设计原则：

- 案件目标围绕“根据方向和位置找到正确地点”。
- 地图只作为轻量背景，不做复杂迷宫。
- Boss 是最终路线推理，不使用战斗语言。
- 错题仍进入悬案馆，反馈保持低压力。

## 9. 技术影响评估

| 模块 | 影响 | 结论 |
| --- | --- | --- |
| `content-schema` | 现有 `QuestionType` 已包含方向题可能需要的 `single`、`judge`、`sort`、`drag` | 首批可不改 schema |
| `content-runtime` | 只读取内容包结构 | 无需改动 |
| `game-core` | 判分仍是标准答案比对 | 首批可不改动 |
| `apps/web` | 若只用 `single` / `judge`，现有答题 UI 可复用 | 首批可复用 |
| 回归脚本 | 现有流程假设可答题和结案 | 首批需保证题型可由脚本完成 |
| 视觉设计 | 方向题可能需要简单地图或格子图 | 可先用文字和简单选项表达 |
