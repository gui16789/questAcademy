## 2. 文档目的

本文档将 `v0.2.0 内容抽取实施计划` 拆分为可直接创建到 GitHub 的 Issue。

v0.2.0 的目标是：

```text
内容抽取和结构化，不新增玩法，不扩展新案件，不接后端。
```

## 3. 已确认关键决策

| 编号 | 决策 | 说明 |

| D1 | v0.2.0 拆成 6 个主 Issue | 与实施计划一致 |

## 5. Issue 1：建立内容包基础


### 目标

建立 `content/math/bsd/grade-2/semester-2/unit-1-division/` 内容包基础结构。

### 范围

新增：

```text
manifest.json
textbook.json
knowledge-map.json
```

### 不做

- 不修改页面逻辑。
- 不抽取题目。
- 不增加内容加载器。
- 不增加校验脚本。

### 交付物

```text
content/math/bsd/grade-2/semester-2/unit-1-division/manifest.json
content/math/bsd/grade-2/semester-2/unit-1-division/textbook.json
content/math/bsd/grade-2/semester-2/unit-1-division/knowledge-map.json
```

### 验收标准

- manifest 字段完整。
- 教材节点包含 4 个节点。
- 知识图谱包含 5 个知识点。
- 能力点和易错点与内容包样例一致。
- ID 命名与 Schema 文档一致。

### 关联文档

- `动物侦探城 - 数据模型与内容 Schema - v0.1.0.md`

## 7. Issue 3：抽取题目和 Boss

```text
标题：content: 抽取题目和 Boss 内容
类型：content
优先级：P1
Milestone：v0.2.0
建议分支：content/3-questions-and-boss
```

### 背景

题目和 Boss 是内容包最核心资产，需要从 `app.js` 抽出，并补齐教材、知识点、能力点、易错点字段。

### 目标

抽取 16 道正式题、10 道备用题、2 道 Boss 题和 BossTask 配置。

### 范围

新增：

```text
questions/level-average-sharing.json
questions/level-division-expression.json
questions/level-with-remainder.json
questions/level-remainder-rule.json
questions/boss-carrot-badge-final.json
```

### 不做

- 不修改 app.js 读取逻辑。

- 16 道正式普通题。
- 10 道备用题。

## 8. Issue 4：增加内容加载器

```text
标题：frontend: 增加 JSON 内容包加载器
类型：frontend
优先级：P1
Milestone：v0.2.0
建议分支：feature/4-content-loader
```

### 背景

内容已抽成 JSON 后，应用需要从内容包加载案件、关卡、题目、知识卡、线索、Boss 和勋章。

### 目标

让当前静态 MVP 从 JSON 内容包读取内容，页面体验基本不变。

### 范围

修改：

```text
app.js
README.md
```

新增或实现：

```text
CONTENT_PACKAGE_PATH
loadJson()
loadContentPackage()
buildRuntimeContent()
内容加载失败提示
```

### 不做

- 不引入 React / Vite。
- 不接后端。
- 不新增玩法。
- 不改变 UI 设计。

### 验收标准

- 应用可通过本地静态服务运行。
- 普通关卡从 JSON 读取题目。
- 线索、知识卡、勋章来自 JSON。
- Boss 来自 JSON。
- 内容加载失败时有友好提示。
- 核心流程与 v0.1.0 一致。

### 关联文档

- `动物侦探城 - 技术方案与工程路线 - v0.1.0.md`
- `动物侦探城 - v0.2.0 内容抽取实施计划.md`

## 9. Issue 5：增加内容校验脚本

```text
标题：test: 增加内容包校验脚本
类型：test / content
优先级：P1
Milestone：v0.2.0
建议分支：test/5-validate-content
```

### 背景

内容包 JSON 增多后，需要校验 ID、引用和必填字段，避免运行时才发现内容错误。

### 目标

新增 Node.js 原生校验脚本。

### 范围

新增：

```text
scripts/validate-content.mjs
```

### 不做

- 不引入 Ajv。
- 不做完整 JSON Schema。
- 不接 GitHub Actions。

### 交付物

命令：

```bash
node scripts/validate-content.mjs
```

### 验收标准

- 校验成功退出码为 0。
- 校验失败退出码非 0。
- 检查 ID 唯一。
- 检查必填字段。
- 检查题目引用知识点、能力点、易错点。
- 检查关卡、线索、知识卡、Boss、复习规则引用完整。
- 错误信息能定位文件和 ID。

### 关联文档

- `动物侦探城 - 数据模型与内容 Schema - v0.1.0.md`
- `动物侦探城 - v0.2.0 内容抽取实施计划.md`

## 10. Issue 6：补数据版本并回归测试

```text
标题：test: 补数据版本并完成 v0.2.0 回归测试
类型：test / frontend
优先级：P0
Milestone：v0.2.0
建议分支：test/6-data-version-regression
```

### 背景

v0.2.0 内容抽取后，本地学习记录需要记录内容包版本，并兼容 v0.1.0 旧数据。

### 目标

补齐本地数据版本字段，并完成 v0.2.0 发布前回归测试。

### 范围

修改：

```text
app.js
README.md
```

补充：

```text
dataVersion: "0.2.0"
contentPackageId
contentVersion
v0.1.0 数据兼容读取
```

### 不做

- 不清空旧用户进度。
- 不接后端。
- 不做账号系统。

### 验收标准

- 新数据写入 `dataVersion`。
- 新数据写入 `contentPackageId`。
- 新数据写入 `contentVersion`。
- v0.1.0 旧数据可读取或安全迁移。
- 刷新后进度不丢失。
- 运行 `node scripts/validate-content.mjs` 通过。
- 核心流程完整跑通。
- 移动端 320px 可用。
- 无 P0 / P1 Bug。
