## 2. 接入范围

本轮在保留既有第一单元“除法”和第二单元“方向与位置”的基础上，补齐北师大版数学二年级下册 3-8 单元 draft 内容包，并通过 `content/registry.json` 按 `locationOrder` 接入数学课程路线。

| 顺序 | 内容包 ID | 单元 | 默认案件 |
| --- | --- | --- | --- |
| 1 | `math.bsd.g2.s2.unit-1-division` | 第一单元：除法 | `case-carrot-badge` |
| 2 | `math.bsd.g2.s2.unit-2-direction-position` | 第二单元：方向与位置 | `case-lost-parcel` |
| 3 | `math.bsd.g2.s2.unit-3-large-numbers` | 第三单元：生活中的大数 | `case-large-number-code` |

## 3. 内容包约定

- 每个新增单元均包含 `manifest.json`、`textbook.json`、`knowledge-map.json`、1 个默认案件、4 个普通关卡题组、1 个 Boss 题组、备用题、知识卡、线索、勋章和错题复习规则。
- 每个普通关卡保持 4 题，答对 3 题通过；Boss 保持 2 道综合题，全部答对结案。
- 题型只使用当前学生端和回归脚本已稳定支持的 `single`、`judge`、`fill`。
- 新增内容包状态为 `draft`，review 状态为 `changes_requested`，题目、剧情、解析、线索和知识卡均为原创草案。

## 4. 技术变更

- `apps/web/src/contentPackage.ts` 改为基于 Vite `import.meta.glob` 自动打包 `content/**/*.json`，避免后续每新增内容包都手动维护 JSON import 清单。
- `scripts/regression-smoke.mjs` 的线索、知识卡和通关数量断言改为按内容包实际关卡数计算。
- 本轮没有扩展后端，没有重写整体 UI，没有改变现有案件、关卡、知识点、错题、勋章流程。

## 5. 待审核事项

- 教研负责人需要复核各单元课时边界、知识点拆解、题目难度和教材页码。
- 内容负责人需要复核案件包装是否适合二年级儿童阅读。
- 儿童测试前不应将新增内容包推进到 `approved` 或 `published`。
