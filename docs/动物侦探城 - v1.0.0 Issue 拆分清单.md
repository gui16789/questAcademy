## 4. Issue 1：搭建工程骨架

```text
标题：frontend: 搭建 v1.0.0 React Vite 工程骨架
类型：feature / frontend
优先级：P0
Milestone：v1.0.0
建议分支：feature/1-vite-react-foundation
```

### 背景

v0.2.0 已完成 JSON 内容包抽取，但应用仍是 `index.html / styles.css / app.js` 静态单页。v1.0.0 需要进入正式前端工程。

### 目标

建立 `React + TypeScript + Vite` 工程骨架和基础 npm scripts。

### 范围

- 新增 `package.json`。
- 新增 `apps/web`。
- 新增 `packages` 基础目录。
- 配置 TypeScript。
- 配置 Vite。
- 保留当前静态 MVP 文件，直到迁移完成。

### 不做

- 不迁移页面。
- 不迁移内容 runtime。
- 不重写游戏逻辑。
- 不新增玩法。

### 验收标准

- `npm install` 可执行。
- `npm run dev` 可启动。
- `npm run build` 可通过。
- 工程目录符合计划。

## 6. Issue 3：建立 content-runtime

```text
标题：architecture: 建立内容包 runtime 加载与查询
类型：feature / architecture
优先级：P1
Milestone：v1.0.0
建议分支：feature/3-content-runtime
```

### 目标

把 v0.2.0 的 JSON 加载和转换逻辑抽到 `packages/content-runtime`。

### 范围

- `loadContentPackage()`。
- 内容引用解析。
- 根据 ID 查询案件、关卡、题目、知识卡、Boss、勋章、复习规则。
- 内容加载失败错误类型。

### 不做

- 不处理 UI。
- 不读写 localStorage。
- 不做后端加载。

### 验收标准

- 可加载当前除法内容包。
- 查询 API 覆盖当前页面需要。
- 单元测试覆盖核心查询。
