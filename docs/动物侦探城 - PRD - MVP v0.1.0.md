## 3. MVP 定位

### 3.1 一句话目标

```text
验证小学二年级孩子是否愿意通过探案闯关方式完成一个数学学习闭环，并愿意回到错题复习和知识点复习。
```

### 3.2 版本边界

| 项目 | 范围 |
| --- | --- |
| 产品阶段 | Phase 0 / MVP 验证 |
| 版本号 | v0.1.0 |
| 学科 | 数学 |
| 教材 | 北师大版小学数学二年级下册 |
| 单元 | 第一单元：除法 |
| 案件 | 徽章失踪案 |
| 实现方式 | 静态单页 |
| 存储方式 | localStorage，失败时退回 session memory |

### 3.3 与后续版本关系

v0.1.0 重点验证闭环。

```text

不扩大内容范围
优先把内容从 app.js 抽成 JSON 内容包
补齐 contentPackageId / contentVersion / dataVersion
```

## 4. 目标用户

### 4.1 核心用户

```text
小学二年级下册学生，年龄约 7-8 岁。
```

### 4.2 使用场景

```text
家庭课后练习为主。
孩子可以独立操作，家长可以旁观或轻度辅助。
单次体验控制在 10-15 分钟。
```

### 4.3 用户特点

- 已具备基础识字能力，但不适合长文本。
- 对动物角色、探案、地图、奖励有兴趣。
- 连续做错题时容易挫败。
- 需要明确下一步操作。
- 不适合复杂系统和高压竞技。

## 5. MVP 核心闭环

```text
进入产品
-> 输入昵称
-> 进入地图
-> 接到徽章失踪案
-> 完成 4 个普通关卡
-> 每关获得 1 条线索
-> 答错题进入悬案馆
-> 解锁线索库知识卡
-> 集齐 4 条线索
-> 挑战 Boss
-> Boss 通过后结案
-> 获得勋章
-> 返回线索库和悬案馆复习
```

## 6. 功能范围

### 6.1 MVP 包含

| 模块 | 优先级 | 说明 |
| --- | --- | --- |
| 开始页 | P0 | 输入昵称，进入产品 |
| 地图页 | P0 | 案件、悬案馆、线索库、勋章馆入口 |
| 案件详情页 | P0 | 展示案件背景、目标和进度 |
| 普通关卡 | P0 | 4 个知识点关卡 |
| 答题系统 | P0 | 单选、判断、填空 |
| 线索获得 | P0 | 通关后获得线索 |
| Boss 挑战 | P0 | 集齐 4 条线索后解锁 |
| 结案页 | P0 | 展示真相、总结和奖励 |
| 悬案馆 | P0 | 错题记录和重做 |
| 线索库 | P0 | 知识卡复习 |
| 勋章馆 | P0 | 成就展示 |
| 本地学习记录 | P0 | 保存进度、错题、勋章 |

### 6.2 MVP 不包含

| 功能 | 不做原因 |
| --- | --- |
| 家长端 | 会增加账号、报告和数据设计复杂度 |
| 老师端 | 涉及班级、作业和学生管理 |
| 排行榜 | 容易增加比较压力 |
| 商城 | 偏离学习目标并带来儿童合规风险 |
| 宠物养成 | 非核心闭环 |
| 多人模式 | 技术和规则复杂 |
| AI 自动出题 | 题目质量和审核风险高 |
| 账号系统 | MVP 先验证本地闭环 |
| 后端 API | MVP 静态验证即可 |
| 内容后台 | v3.0 再考虑 |

### 7.2 知识点

| 顺序 | 知识点 | 关卡 |
| --- | --- | --- |
| 1 | 平均分 | 仓库平均分 |

## 8. 页面需求

### 8.1 开始页

目标：

```text
让孩子进入动物侦探城并创建本地昵称。
```

页面元素：

- 产品名称。
- 简短引导语。
- 昵称输入框。
- 开始按钮。

验收标准：

- 用户可输入昵称。
- 未输入时使用默认昵称“小侦探”。
- 点击开始后进入地图页。
- 用户开始状态被保存。

### 8.2 地图页

目标：

```text
展示核心入口和当前调查进度。
```

入口：

- 徽章失踪案。
- 悬案馆。
- 线索库。
- 勋章馆。
- Boss 挑战，未解锁时禁用。

验收标准：

- 可进入案件详情。
- 可进入悬案馆、线索库、勋章馆。
- Boss 未解锁时不可进入。
- 已结案后显示结案状态。

### 8.3 案件详情页

目标：

```text
让孩子理解发生了什么、要调查什么、当前进度如何。
```

页面元素：

- 案件名称。
- 案件背景。
- 调查目标。
- 线索进度。
- 关卡列表。
- 开始调查按钮。

验收标准：

- 首次进入看到案件背景。
- 点击开始后进入当前可玩关卡。
- 已通过关卡显示完成状态。
- 未解锁关卡不可进入。

### 8.4 普通关卡答题页

目标：

```text
完成单个知识点练习。
```

页面元素：

- 关卡名。
- 题号进度。
- 知识点标签。
- 题干。
- 选项或数字输入。
- 提交按钮。
- 反馈区。
- 下一题 / 查看结果按钮。

验收标准：

- 可展示题目。
- 可选择或输入答案。
- 未作答时不可提交。
- 提交后显示对错和解析。
- 答错进入悬案馆。
- 完成本关后按通过规则判断结果。

### 8.5 线索获得页

目标：

```text
给孩子阶段完成感，并推动继续调查。
```

验收标准：

- 通过关卡后展示对应线索。
- 线索保存到进度。
- 对应知识卡解锁。
- 可继续下一关或进入线索库。

### 8.6 Boss 挑战页

目标：

```text
综合验证本案知识点。
```

规则：

- 集齐 4 条线索后解锁。
- 包含 2 道 Boss 步骤题。
- 2 步都答对才结案。
- 任一步答错则进入失败诊断。

验收标准：

- 未集齐线索不能进入 Boss。
- Boss 错题进入悬案馆。
- Boss 失败后可重试或去线索库复习。
- Boss 通过后进入结案页。

### 8.7 结案页

目标：

```text
展示案件真相、学习成果和奖励。
```

页面元素：

- 已结案状态。
- 案件真相。
- 收集线索数量。
- 错题数量。
- 获得勋章。
- 去线索库。
- 去悬案馆。
- 返回地图。

验收标准：

- Boss 通过后自动结案。
- 发放“破案小能手”勋章。
- 结案状态保存。
- 刷新后不丢失。

### 8.8 悬案馆

目标：

```text
把错题复习包装成重新调查。
```

验收标准：

- 答错题自动进入悬案馆。
- 展示未侦破和已侦破状态。
- 可重新作答错题。
- 重新答对后标记为已侦破。
- 首次进入悬案馆发放“悬案调查员”勋章。

### 8.9 线索库

目标：

```text
让知识复习像查看侦探线索。
```

验收标准：

- 通过关卡后解锁对应知识卡。
- 未解锁知识卡显示锁定或弱化状态。
- 知识卡展示解释、例题、方法、常见错误。
- 可从地图和结案页进入。

### 8.10 勋章馆

目标：

```text
展示孩子的学习成就。
```

验收标准：

- 达成条件后自动发放勋章。
- 勋章不重复发放。
- 已获得和未获得状态清楚。
- 展示获得时间和原因。

## 9. 规则需求

### 9.1 普通关卡规则

```text
每关 4 道题。
答对至少 3 道通过。
答错不阻断继续答题。
通过后获得 1 条线索。
通过后解锁 1 张知识卡。
```

### 9.2 Boss 规则

```text
集齐 4 条线索后解锁。
2 道 Boss 步骤题都答对才结案。
答错进入悬案馆。
失败后可以重试，也可以回线索库复习。
```

### 9.3 错题规则

```text
普通关卡和 Boss 答错题进入悬案馆。
同一道题多次答错，错误次数增加。
重新答对后状态变为已侦破。
```

### 9.4 勋章规则

| 勋章 | 获得条件 |
| --- | --- |
| 新手小侦探 | 完成第一个普通关卡 |
| 线索收集员 | 收集全部 4 条线索 |
| 悬案调查员 | 第一次进入悬案馆复习 |
| 坚持不放弃 | 错题重做并答对 |
| 破案小能手 | 完成 Boss 并结案 |

## 10. 数据需求

### 10.1 v0.1.0 存储方式

```text
localStorage
```

当 localStorage 不可用时：

```text
退回 session memory
```

v0.1.0 不支持：

- 账号登录。
- 跨设备同步。
- 云端保存。
- 家长远程查看。

### 10.2 需要保存的数据

- 用户昵称。
- 是否已开始。
- 当前关卡。
- 已通过关卡。
- 已收集线索。
- Boss 是否解锁。
- 案件是否结案。
- 答题记录。
- 错题记录。
- 勋章记录。

### 10.3 v0.2.0 数据补充

v0.2.0 起需要补充：

```text
contentPackageId
contentVersion
dataVersion
knowledgePointId
skillId
misconceptionId
```

## 11. 交互与反馈文案

### 11.1 答对

示例：

```text
太棒了！你找到了一条新线索。
你的推理很准确，继续调查吧！
```

### 11.2 答错

示例：

```text
这条线索还不够清楚，我们先放进悬案馆，稍后再调查。
别着急，小侦探再想一想。
```

禁用表达：

```text
失败
太差了
错误太多
排名落后
```

### 11.3 Boss 失败

示例：

```text
Boss 的谜题还没有完全破解。
你可以先去线索库看看提示，也可以马上再挑战一次。
```

## 12. 验收标准

### 12.1 核心流程验收

必须全部通过：

- 可从开始页进入地图。
- 可从地图进入案件。
- 可完成 4 个普通关卡。
- 每关通过后获得线索。
- 答错题进入悬案馆。
- 知识卡随关卡解锁。
- 集齐 4 条线索后 Boss 解锁。
- Boss 通过后进入结案页。
- 结案后发放勋章。
- 刷新后进度不丢失。

### 12.2 内容验收

- 题目答案正确。
- 解析准确。
- 知识点与题目匹配。
- 文案适合二年级学生。
- Boss 是综合应用，不是单纯变难。

### 12.3 UI 验收

- 移动端可用。
- 主要按钮清晰。
- 题目可读。
- 反馈温和。
- 未解锁状态清楚。
- 勋章和线索状态清楚。

### 12.4 Bug 等级

| 等级 | 定义 |
| --- | --- |
| P0 | 阻断核心流程，如无法答题、无法进入案件 |
| P1 | 严重影响体验或数据，如错题不保存、Boss 无法结案 |
| P2 | 一般问题，如文案错字、样式轻微错位 |
| P3 | 轻微优化，如动效和装饰 |

MVP 发布前不得存在 P0 / P1 Bug。

## 14. 测试计划

### 14.1 测试对象

```text
5-10 名小学二年级下册或接近该水平的孩子。
```

### 14.2 测试方式

- 家庭或现场测试。
- 单次体验 10-15 分钟。
- 尽量观察孩子独立操作。
- 家长可旁观但不直接提示。
- 测试后分别收集孩子和家长反馈。

### 14.3 观察问题

- 孩子是否能独立进入案件。
- 孩子是否知道下一步点哪里。
- 哪些页面出现停顿。
- 答错后是否愿意继续。
- 是否主动查看线索库。
- 是否愿意进入悬案馆。
- 获得勋章时是否有成就感。
- 家长是否认为有学习价值。
