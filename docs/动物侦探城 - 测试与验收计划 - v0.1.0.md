## 2. 文档目的

本文档定义《动物侦探城》v0.1.0 MVP 的测试范围、测试类型、验收标准、Bug 等级、儿童测试方法、上线前检查和阶段门规则。

MVP 测试不只判断“功能是否能点”，还要判断：

- 孩子是否能独立理解流程。
- 题目和解析是否准确。
- 错题复习是否真的被使用。
- Boss 难度是否合理。
- 家长是否认可学习价值。

## 3. 已确认关键决策

| 编号 | 决策 | 说明 |
| --- | --- | --- |
| D1 | 测试分为 6 类 | 功能、内容、学习闭环、儿童体验、兼容性、数据保存 |
| D2 | 上线前必须无 P0 / P1 Bug | 阻断和严重数据问题不可发布 |
| D3 | 儿童测试样本为 5-10 名孩子 | 年龄 7-8 岁或接近二年级下册水平 |
| D4 | 儿童测试记录行为、学习表现、主观反馈 | 不只记录分数 |
| D5 | Bug 等级采用 P0-P3 | 与项目规范一致 |
| D6 | v0.2 和 v1.0 阶段门分开 | 可先结构化，不盲目进入正式扩展 |

## 4. 测试分层

### 4.1 功能测试

验证页面、按钮、答题、解锁、跳转、反馈是否按 PRD 实现。

### 4.2 内容测试

验证题目、答案、解析、知识点、线索、知识卡和 Boss 是否准确、适龄、符合教材。

### 4.3 学习闭环测试

验证从接案到结案、错题复习、知识卡复习、勋章奖励的完整闭环是否成立。

### 4.4 儿童体验测试

验证孩子是否看得懂、愿意继续、答错后是否低压力、是否愿意复习错题。

### 4.5 兼容性测试

验证手机、平板、桌面视口下可用，尤其是 320px 宽度移动端。

### 4.6 数据保存测试

验证刷新后进度、错题、线索、知识卡和勋章不丢失。

## 5. 上线前硬性门槛

MVP 可测试 / 可发布前必须满足：

```text
核心流程完整跑通
无 P0 / P1 Bug
题目答案和解析已检查
错题能进入悬案馆并重新侦破
线索和知识卡能正确解锁
Boss 能正确解锁、失败、重试、结案
刷新后进度不丢失
移动端 320px 宽度可用
```

如果任一项不满足，不进入儿童测试。

## 6. 功能测试清单

### 6.1 开始页

| 检查项 | 预期 |
| --- | --- |
| 输入昵称并开始 | 进入地图页 |
| 不输入昵称 | 使用默认昵称“小侦探” |
| 刷新后 | 保留已开始状态 |

### 6.2 地图页

| 检查项 | 预期 |
| --- | --- |
| 点击案件入口 | 进入案件详情页 |
| 点击悬案馆 | 进入错题列表 |
| 点击线索库 | 进入知识卡列表 |
| 点击勋章馆 | 进入勋章列表 |
| Boss 未解锁时点击 | 不可进入 |
| 结案后返回地图 | 显示已结案状态 |

### 6.3 普通关卡

| 检查项 | 预期 |
| --- | --- |
| 进入第 1 关 | 展示第 1 关题目 |
| 未选择答案 | 提交按钮不可用 |
| 答对 | 显示正确反馈和解析 |
| 答错 | 显示温和反馈，错题进入悬案馆 |
| 完成 4 题且答对 >= 3 | 通过关卡 |
| 完成 4 题且答对 < 3 | 不通过，提示可复习 |
| 通过关卡 | 获得线索并解锁知识卡 |

### 6.4 Boss

| 检查项 | 预期 |
| --- | --- |
| 未集齐 4 条线索 | Boss 不可进入 |
| 集齐 4 条线索 | Boss 解锁 |
| Boss 2 题都答对 | 进入结案页 |
| Boss 任一题答错 | 进入失败反馈，可重试或复习 |
| Boss 错题 | 进入悬案馆 |

### 6.5 悬案馆

| 检查项 | 预期 |
| --- | --- |
| 答错题出现 | 状态为未侦破 |
| 重做答对 | 状态变为已侦破 |
| 重做答错 | 错误次数增加 |
| 第一次进入悬案馆 | 发放悬案调查员勋章 |

### 6.6 线索库

| 检查项 | 预期 |
| --- | --- |
| 未通过关卡 | 对应知识卡锁定或弱化 |
| 通过关卡 | 对应知识卡解锁 |
| 点击知识卡 | 展示解释、例题、方法、常见错误 |

### 6.7 勋章馆

| 检查项 | 预期 |
| --- | --- |
| 达成条件 | 自动发放勋章 |
| 重复达成条件 | 不重复发放 |
| 未获得勋章 | 显示未获得状态 |
| 已获得勋章 | 显示获得时间和原因 |

## 7. 内容测试清单

| 检查项 | 标准 |
| --- | --- |
| 教材一致性 | 符合北师大版二年级下册第一单元除法 |
| 知识点准确性 | 没有概念错误 |
| 答案准确性 | 每道题答案正确 |
| 解析准确性 | 解析能解释为什么 |
| 难度适龄 | 适合二年级学生 |
| 语言可读性 | 短句、少抽象词 |
| 错题提示 | 温和，不制造挫败 |
| Boss 合理性 | 综合应用，不是单纯变难 |
| 知识卡有效性 | 能帮助孩子复习 |
| 线索关联性 | 与案件和知识点有关 |

## 8. 学习闭环测试

完整闭环必须跑通：

```text
开始页
-> 地图页
-> 案件详情
-> 第 1 关
-> 获得线索
-> 第 2 关
-> 获得线索
-> 第 3 关
-> 获得线索
-> 第 4 关
-> 获得线索
-> Boss 解锁
-> Boss 挑战
-> 结案
-> 勋章
-> 线索库复习
-> 悬案馆复习
```

验收标准：

- 用户能完成完整闭环。
- 每个状态都能保存。
- 每个入口都能返回地图或继续流程。
- 错题和知识卡能在闭环后继续使用。

## 9. 数据保存测试

### 9.1 必测场景

| 场景 | 预期 |
| --- | --- |
| 输入昵称后刷新 | 昵称保留 |
| 通过关卡后刷新 | 关卡进度保留 |
| 获得线索后刷新 | 线索保留 |
| 答错后刷新 | 错题保留 |
| 错题侦破后刷新 | 已侦破状态保留 |
| 获得勋章后刷新 | 勋章保留 |
| 结案后刷新 | 结案状态保留 |

### 9.2 localStorage 不可用

如果 localStorage 不可用：

- 当前 session 内仍可体验。
- 刷新后可以丢失数据。
- 不应阻断答题流程。

## 11. 儿童体验测试

### 11.1 测试对象

```text
5-10 名孩子
年龄 7-8 岁或接近二年级下册水平
```

### 11.2 测试方式

- 单次体验 10-15 分钟。
- 尽量让孩子独立操作。
- 家长旁观，但不直接提示。
- 观察员记录行为。
- 结束后询问孩子和家长。

### 11.3 行为观察

记录：

- 是否能自己开始。
- 是否知道下一步点哪里。
- 哪个页面停顿。
- 是否读得懂题目。
- 答错后是否愿意继续。
- 是否主动查看线索库。
- 是否愿意进入悬案馆。
- 获得勋章是否有正反馈。

### 11.4 学习表现

记录：

- 普通关卡通过情况。
- 错题数量。
- 错题类型。
- Boss 是否通过。
- 是否重做错题。
- 是否查看知识卡。

### 11.5 主观反馈

问孩子：

```text
你知道自己在帮谁破案吗？
你觉得题目难不难？
答错的时候会不会不开心？
你想不想继续玩下一案？
你喜不喜欢拿到线索和勋章？
```

问家长：

```text
你觉得这种形式有学习价值吗？
孩子是否比普通练习更愿意做？
题目难度是否合适？
你能看出孩子错在哪里吗？
你愿意让孩子下次继续使用吗？
```

## 12. Bug 等级

| 等级 | 定义 | 示例 | 发布要求 |
| --- | --- | --- | --- |
| P0 | 阻断核心流程 | 无法进入案件、无法答题、无法结案 | 必须修复 |
| P1 | 严重影响体验或数据 | 错题不保存、Boss 无法解锁、刷新丢进度 | 必须修复 |
| P2 | 一般问题 | 文案错字、样式轻微错位、非核心按钮异常 | 可评估 |
| P3 | 轻微优化 | 动效、装饰、细节体验 | 可后置 |

MVP 发布前不得存在 P0 / P1。

### 13.1 v1.0.0 自动回归命令

v1.0.0 工程化版本发布前，除基础命令外，需要运行：

```bash
npm run test:regression
```

该命令会启动 `apps/web` 本地 Vite 服务，并使用本机 Chrome headless 执行：

- 4 个普通关卡完整答题。
- Boss 解锁和结案。
- v0.2.0 localStorage 迁移到 v1.0.0。
- 刷新后进度、错题、勋章保留。
- 320px 移动端无横向溢出。

如果本机 Chrome 路径不在默认位置，可通过 `CHROME_PATH` 指定。

## 15. 阶段门规则

### 15.1 进入 v0.2.0 的条件

进入 v0.2.0 的目标是做内容抽取和结构化。

条件：

- 静态 MVP 核心流程可用。
- 题目和内容基本正确。
- 无 P0 / P1 Bug。
- 项目值得继续结构化。
- 已明确内容包、Schema 和工程路线。

说明：

```text
即使儿童测试指标未完全达标，也可以进入 v0.2.0 做结构化。
但 v0.2.0 不应扩展新玩法。
```

### 15.2 进入 v1.0 的条件

进入 v1.0 的目标是正式打磨首个单元体验。

条件：

- 儿童测试完成。
- 案件完成率达到目标或接近目标。
- Boss 难度可接受。
- 家长认可学习价值。
- 错题复习机制有使用迹象。
- 内容生产流程可复制。
- 技术结构已支持内容包。

说明：

```text
如果儿童测试显示孩子无法理解主流程，不进入 v1.0 扩展。
应先修正流程、文案、题目或交互。
```
