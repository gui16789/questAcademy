## 2. 文档目的

本文档定义《动物侦探城》的游戏世界观、角色体系、案件结构、线索规则、Boss 规则、失败反馈、奖励成长和地图扩展方式。

它用于约束后续内容扩展，避免产品在以下方向跑偏：

- 变成普通刷题工具。
- 变成纯剧情游戏。
- 变成高压考试。
- 变成暗黑成人侦探风。
- 变成重度养成或商城系统。

## 3. 已确认关键决策

| 编号 | 决策 | 说明 |
| --- | --- | --- |
| D1 | 世界观是明亮、友好、低压力的动物城市学习世界 | 儿童友好优先 |
| D2 | 探案不涉及恐怖、暴力、真实犯罪、羞辱式失败 | 只做找线索、查记录、破解谜题 |
| D3 | MVP 角色固定为小侦探、鹿队长、兔仓管、狐狸配送员、熊老师 | 后续按功能扩展角色 |
| D4 | 标准案件结构固定 | 接案、关卡、线索、Boss、结案、总结、奖励、复习 |
| D5 | 失败反馈低压力表达 | 答错是未侦破，不是失败 |
| D6 | MVP 只做 5 枚勋章 | 不做复杂等级、商城、宠物 |
| D7 | 地图按版本逐步扩展 | 从单案件到多单元、多学科城市 |

## 4. 世界观定位

### 4.1 一句话世界观

```text
动物侦探城是一座明亮、友好、充满学习谜题的动物城市，孩子作为动物侦探学院的新手小侦探，通过数学调查、线索收集和最终推理完成一个个学习案件。
```

### 4.2 世界观原则

- 城市是温暖的，不是阴暗的。
- 侦探是学习身份，不是真实执法身份。
- 案件是学习谜题，不是真实犯罪。
- Boss 是最终推理任务，不是敌人战斗。
- 错题是待调查线索，不是失败惩罚。

### 4.3 学习与游戏映射

| 学习概念 | 游戏表达 |
| --- | --- |
| 教材单元 | 城市区域 / 案件组 |
| 知识点 | 调查关卡 |
| 题目 | 调查任务 |
| 答对 | 找到线索 |
| 答错 | 线索还不清楚 |
| 错题本 | 悬案馆 |
| 知识卡 | 线索库 |
| 综合应用 | Boss 最终推理 |
| 单元完成 | 结案 |
| 学习成就 | 勋章 |

## 5. 探案表达边界

### 5.1 禁止方向

不得出现：

- 恐怖。
- 暴力。
- 真实犯罪。
- 血迹。
- 审讯。
- 惩罚。
- 失败羞辱。
- 阴暗成人侦探风。
- 过度警察化表达。

### 5.2 推荐表达

推荐使用：

- 找线索。
- 查记录。
- 对说法。
- 整理档案。
- 破解谜题。
- 重新调查。
- 完成结案。
- 收集勋章。

### 5.3 语言替代表

| 避免使用 | 推荐替代 |
| --- | --- |
| 失败 | 还没完全破解 |
| 错题 | 未侦破悬案 |
| 惩罚 | 再调查一次 |
| 敌人 | 最终谜题 |
| 打败 Boss | 完成最终推理 |
| 犯人 | 线索对象 / 说法不合理的角色 |

## 6. 角色体系

### 6.1 MVP 角色

| 角色 | 定位 | 功能 |
| --- | --- | --- |
| 小侦探 | 孩子扮演的主角 | 学习、答题、收集线索、结案 |
| 鹿队长 | 导师 / 引导者 | 说明任务、鼓励、提示下一步 |
| 兔仓管 | 案件角色 | 提供仓库平均分线索 |
| 狐狸配送员 | 案件角色 | 提供分发记录和余数相关线索 |
| 熊老师 | 案件角色 | 提供最终记录、结案总结和奖励 |

### 6.2 角色功能分类

后续角色按功能分类：

| 类型 | 作用 |
| --- | --- |
| 导师角色 | 引导、提示、总结、鼓励 |
| 案件角色 | 提供线索、情境和说法 |
| 地图区域角色 | 标识不同单元区域 |
| 奖励角色 | 颁发勋章或总结成长 |

### 6.3 角色设计规则

- 原创动物角色。
- 不复刻已有商业 IP。
- 不使用恐怖、凶狠、压迫性形象。
- 角色语言短句化。
- 角色不替孩子做题，只引导孩子思考。

## 7. 标准案件结构

### 7.1 案件流程

```text
接案
-> 案件背景
-> 普通调查关卡
-> 每关获得线索
-> 线索集齐
-> Boss 最终推理
-> 结案真相
-> 学习总结
-> 勋章奖励
-> 复习入口
```

### 7.2 案件模板

```text
案件 ID：
案件名称：
所属学科：
教材版本：
年级学期：
单元：
核心知识点：
案件背景：
导师角色：
案件角色：
地图区域：
关卡列表：
线索列表：
Boss 任务：
结案真相：
学习总结：
奖励勋章：
复习入口：
```

### 7.3 案件设计原则

- 一个案件聚焦一个单元或一个知识主题。
- 每个普通关卡对应一个核心知识点。
- 每条线索必须和知识点有关。
- Boss 必须综合多个知识点。
- 结案页必须回到学习总结。

## 8. 关卡设计

### 8.1 关卡定位

关卡是知识点练习的游戏化包装。

```text
知识点 -> 关卡
能力点 -> 题组
易错点 -> 错题分类
```

### 8.2 关卡模板

```text
关卡 ID：
关卡名称：
所属案件：
场景：
关联知识点：
关联能力点：
题目数量：
通过条件：
答对反馈：
答错反馈：
通关线索：
解锁知识卡：
可能错因：
```

### 8.3 MVP 关卡

| 顺序 | 关卡 | 知识点 | 场景 |
| --- | --- | --- | --- |
| 1 | 仓库平均分 | 平均分 | 兔仓管的徽章仓库 |
| 2 | 分发记录表 | 除法算式含义 | 学院分发台 |
| 3 | 剩下的徽章 | 有余数除法 | 运输路线口 |
| 4 | 可疑的说法 | 余数小于除数 | 熊老师办公室 |

## 9. 线索规则

### 9.1 线索定位

线索是完成知识点关卡后的游戏化反馈。

线索不是装饰，必须推动：

- 案件进展。
- Boss 解锁。
- 知识卡解锁。
- 学习总结。

### 9.2 线索规则

```text
通过普通关卡
-> 获得 1 条案件线索
-> 解锁 1 张知识卡
-> 更新线索进度
```

### 9.3 MVP 线索

| 关卡 | 线索 |
| --- | --- |
| 仓库平均分 | 仓库里的徽章原本可以平均分给 4 个侦探小队。 |
| 分发记录表 | 分发记录里有一组除法算式写错了。 |
| 剩下的徽章 | 剩下的 3 枚徽章不是丢了，而是没有组成完整一队。 |
| 可疑的说法 | 狐狸配送员说的余数太大，不符合除法规则。 |

## 10. Boss 最终推理

### 10.1 Boss 定位

Boss 是最终推理任务，不是战斗对象。

Boss 用于验证：

- 是否能综合使用多个知识点。
- 是否理解线索之间的关系。
- 是否能在情境中解释答案。

### 10.2 Boss 设计规则

```text
集齐线索后解锁
绑定多个知识点
绑定多个能力点
包含分步题组
答错后进入失败诊断
失败后推荐复习路径
通过后结案
```

### 10.3 MVP Boss

Boss 名称：

```text
胡萝卜徽章最终推理
```

核心问题：

```text
学院原来有 35 枚胡萝卜徽章，每队分 6 枚。
可以分给几个完整小队，还剩几枚？
狐狸配送员说剩下的徽章还能再分给一个完整小队，这句话对吗？
```

通过规则：

```text
2 步都答对才结案。
```

失败反馈：

```text
最终谜题还没有完全破解。
你可以先去线索库看看提示，也可以再调查一次。
```

## 11. 失败反馈机制

### 11.1 核心原则

```text
答错不是失败，而是线索还没查清楚。
```

### 11.2 状态命名

| 学习状态 | 游戏表达 |
| --- | --- |
| 答错 | 线索还不清楚 |
| 错题 | 未侦破悬案 |
| 重做错题 | 重新调查 |
| 重做答对 | 成功侦破 |
| Boss 未通过 | 最终谜题还没完全破解 |

### 11.3 文案示例

答错：

```text
这条线索还不够清楚，我们先放进悬案馆，稍后再调查。
```

重做答对：

```text
悬案已侦破！这条线索现在清楚了。
```

Boss 未通过：

```text
最终谜题还没有完全破解。回线索库看看提示，再来推理一次吧。
```

## 12. 成长与奖励

### 12.1 奖励原则

奖励必须绑定学习行为。

不得把奖励主要绑定到：

- 登录天数。
- 付费。
- 排名。
- 装饰收集。
- 无学习意义的点击。

### 12.2 MVP 勋章

| 勋章 | 获得条件 |
| --- | --- |
| 新手小侦探 | 完成第一个普通关卡 |
| 线索收集员 | 收集全部 4 条线索 |
| 悬案调查员 | 第一次进入悬案馆复习 |
| 坚持不放弃 | 错题重做并答对 |
| 破案小能手 | 完成 Boss 并结案 |

### 12.3 后续成长方向

后续可以评估：

- 单元徽章。
- 知识点掌握徽章。
- 复习坚持徽章。
- 多案件结案记录。
- 学习报告中的成长轨迹。

但必须避免：

- 商城化。
- 抽卡化。
- 排名刺激。
- 复杂宠物养成抢走学习重点。

## 13. 地图扩展

### 13.1 阶段路线

| 阶段 | 地图形态 |
| --- | --- |
| v0.x | 一个案件入口 |
| v1.x | 除法区域多个案件 |
| v1.5 | 方向与位置新区域 |
| v2.0 | 二年级下册多单元地图 |
| v3.0 | 多学科 / 多年级学习城市 |

### 13.2 地图原则

- 地图是学习路径，不是复杂迷宫。
- 入口状态必须清楚。
- 未解锁区域要有原因。
- 已完成区域要有结案标识。
- 每个区域对应明确教材单元或知识主题。

## 15. MVP 游戏设计验收

MVP 必须满足：

- 孩子知道自己是小侦探。
- 孩子知道当前案件目标。
- 每关有明确调查任务。
- 每关通关后有线索反馈。
- 错题进入悬案馆而不是失败列表。
- Boss 是最终推理，不是战斗。
- 结案页有学习总结。
- 勋章奖励与学习行为绑定。
- 整体氛围明亮、友好、低压力。

## 16. 待确认问题

1. 后续是否需要为每个单元固定一个地图区域风格。
2. 角色是否需要建立独立角色设定表。
3. 勋章是否需要统一视觉等级，还是保持无稀有度。
4. Boss 是否统一命名为“最终推理”，避免使用战斗语言。
5. 多单元地图是否采用线性解锁，还是允许自由选择。
