# 动物侦探城：UI 视觉设计规范

## 1. 文档目的

本文档用于规范《动物侦探城》项目的 UI 视觉设计方向、组件体系、页面布局、移动端适配、图片资产分层和设计交付标准。

本项目不是普通刷题网站，也不是教育后台，而是面向小学低年级儿童的**探案式游戏化学习产品**。因此 UI 设计必须同时满足三件事：

```
像儿童探案绘本一样有趣
像学习工具一样清楚
像游戏系统一样可扩展
```

本文档适用于：

- MVP 原型设计
- V1.0 正式版设计
- 后续地图、案件、关卡、线索卡、勋章和内容扩展
- UI/UX、插画、美术、前端开发、内容策划协作

---

## 2. UI 设计总原则

### 2.1 一句话设计原则

```
用童趣绘本化的视觉包装探案学习，用清晰稳定的组件保证孩子能顺利完成数学闯关。
```

### 2.2 核心体验目标

| 目标 | 说明 |
| --- | --- |
| 童趣 | 让孩子觉得像进入动物侦探学院，而不是打开刷题网站 |
| 清晰 | 题目、按钮、反馈必须一眼能看懂 |
| 低压力 | 答错不制造挫败，而是包装成“悬案待调查” |
| 可扩展 | 后续可持续增加地图、案件、关卡、勋章 |
| 可复用 | 按钮、卡片、弹窗、导师对白等组件统一 |
| 移动优先 | 手机、平板、电脑都能稳定使用 |

### 2.3 设计优先级

| 优先级 | 设计目标 |
| --- | --- |
| 第一优先级 | 儿童能看懂 |
| 第二优先级 | 学习内容清晰 |
| 第三优先级 | 主流程顺畅 |
| 第四优先级 | 探案氛围明确 |
| 第五优先级 | 美术表现精致 |
| 第六优先级 | 动效和装饰丰富 |

### 2.4 不做原则

MVP 和 V1.0 阶段避免以下设计方向：

- 成人化后台风格
- 普通官网风格
- 暗黑侦探风
- 高压考试风
- 复杂 RPG 游戏界面
- 强竞技排行榜视觉
- 信息密度过高的仪表盘
- 大面积深色背景
- 文字直接压在复杂插画上
- 按钮、文字、角色全部合成在一张图里

## 3. 视觉风格规范

### 3.1 视觉关键词

```
童趣警探学院
动物城市
绘本感
轻探案
地图冒险
线索收集
知识闯关
温暖鼓励
低压力成长
```

### 3.2 风格定位

《动物侦探城》的 UI 应该像一本可互动的儿童探案绘本：

- 有动物角色；
- 有城市地图；
- 有案件任务；
- 有线索、档案、徽章；
- 有清晰的答题卡片；
- 有温和的导师引导；
- 有通关和成长反馈。

### 3.3 风格对照

| 应该像 | 不应该像 |
| --- | --- |
| 儿童探案绘本 | 成人办公后台 |
| 动物城市冒险地图 | 普通教育官网 |
| 侦探学院任务卡 | 冷冰冰的试卷系统 |
| 温暖鼓励的学习游戏 | 高压考试系统 |
| 清晰卡片式界面 | 信息密集的数据看板 |

### 3.4 情绪基调

| 场景 | 情绪 |
| --- | --- |
| 接案 | 好奇、期待 |
| 普通关卡 | 专注、轻松 |
| 答对 | 鼓励、成就 |
| 答错 | 温和、可继续 |
| 悬案馆 | 神秘但不恐怖 |
| Boss | 挑战但不压迫 |
| 结案 | 成就、开心 |
| 勋章馆 | 收集、荣耀 |

## 4. 品牌与世界观视觉

### 4.1 世界观基调

```
动物侦探城是一座明亮、友好、充满学习任务的动物城市。
孩子是动物侦探学院的新手小侦探。
每个知识单元都是一个案件。
每个关卡都是一次调查。
每道错题都是一个待破解的悬案。
```

### 4.2 视觉元素池

| 元素 | 用途 |
| --- | --- |
| 放大镜 | 查找线索、提示、搜索 |
| 档案袋 | 案件、错题、记录 |
| 侦探徽章 | 成就、身份、奖励 |
| 地图路牌 | 关卡入口、区域入口 |
| 线索板 | Boss 前推理、案件总结 |
| 胡萝卜徽章 | 首个案件核心物品 |
| 印章 | 已结案、已侦破、已掌握 |
| 便签纸 | 知识卡、提示卡 |
| 警探帽 | 主角身份识别 |
| 脚印 | 路径、调查进度 |

### 4.3 视觉危险边界

不能出现：

- 恐怖、阴暗、惊吓元素；
- 真正犯罪、暴力、血迹等成人侦探元素；
- 过强输赢刺激；
- 让孩子羞愧的错误反馈；
- 过度拟真警察系统；
- 版权风险明显的已有 IP 形象。

## 5. 色彩规范

### 5.1 色彩设计原则

```
主色温暖，辅助色明亮，状态色柔和，奖励色突出，整体不刺眼。
```

儿童产品不能依赖大量高饱和颜色堆叠。每个页面建议控制在：

```
1 个主色
1～2 个辅助色
少量状态色
大面积柔和底色
```

### 5.2 核心色板

| 类型 | 名称 | 色值 | 用途 |
| --- | --- | --- | --- |
| 主色 | 胡萝卜橙 | #FF8A3D | 主按钮、核心入口、品牌识别 |
| 主色深 | 探案橙 | #E96F22 | 按钮按下态、重点强调 |
| 主色浅 | 奶油橙 | #FFE1C7 | 浅色背景、卡片点缀 |
| 辅助色 1 | 天空蓝 | #5BB9F0 | 地图、信息提示、导航 |
| 辅助色 2 | 草地绿 | #68C77C | 答对、已完成、知识掌握 |
| 辅助色 3 | 奶油黄 | #FFF3C4 | 档案纸、知识卡、背景 |
| 奖励色 | 徽章金 | #FFC83D | 勋章、通关奖励、结案高光 |
| 错误色 | 柔和红 | #F66B6B | 答错、未侦破、警示 |
| 警示浅色 | 浅红底 | #FFE3E3 | 错题卡背景 |
| 正文色 | 深灰蓝 | #2F3A45 | 正文、题干 |
| 次级文字 | 中灰蓝 | #6B7A8A | 说明、辅助信息 |
| 边框色 | 浅灰蓝 | #D9E2EC | 卡片边框、分割线 |
| 页面底色 | 米白 | #FFF9EF | 主背景 |

### 5.3 状态色规则

| 状态 | 颜色 | 使用场景 |
| --- | --- | --- |
| 默认 | 胡萝卜橙 | 开始调查、提交答案 |
| 成功 | 草地绿 | 答对、已完成、已侦破 |
| 错误 | 柔和红 | 答错、未侦破 |
| 奖励 | 徽章金 | 勋章、通关、结案 |
| 禁用 | 浅灰蓝 | 未解锁、不可点击 |
| 信息 | 天空蓝 | 提示、说明、导航 |

### 5.4 色彩使用禁忌

- 不使用大面积纯黑背景。
- 不使用刺眼荧光色。
- 错误提示不要使用强烈血红。
- 不用颜色作为唯一信息提示，需配合图标或文字。
- 题目区域不要使用复杂渐变背景。
- 奖励金色不要大面积铺满，以免降低重点。

## 6. 字体与文字层级规范

### 6.1 字体原则

```
大字号、高对比、短句、少字、重点突出。
```

### 6.2 推荐字体

| 使用环境 | 推荐 |
| --- | --- |
| 中文默认 | 系统无衬线字体 |
| iOS | PingFang SC |
| Android | Noto Sans SC / 系统默认 |
| Web | system-ui, -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif |

### 6.3 字体层级

| 类型 | 用途 | 移动端字号 | 桌面端字号 | 字重 |
| --- | --- | --- | --- | --- |
| H1 | 页面主标题、案件名 | 26～32px | 32～40px | 700 |
| H2 | 模块标题、区域名 | 22～26px | 28～32px | 700 |
| H3 | 卡片标题、关卡名 | 18～22px | 22～26px | 600 |
| 剧情文本 | 导师对白、案件描述 | 16～18px | 18～20px | 400 / 500 |
| 题目正文 | 数学题干 | 18～22px | 22～26px | 600 |
| 选项文本 | A/B/C/D 选项 | 18～20px | 20～22px | 500 |
| 按钮文字 | 主要操作 | 18～20px | 20～22px | 700 |
| 提示文本 | 答错提示、解析 | 14～16px | 16～18px | 400 / 500 |
| 标签文本 | 状态、难度、已解锁 | 12～14px | 14～16px | 500 |

### 6.4 文字长度规范

| 文案类型 | 建议长度 |
| --- | --- |
| 按钮 | 2～6 个字 |
| 标题 | 4～12 个字 |
| 导师对白 | 每屏不超过 2～3 行 |
| 题目题干 | 尽量不超过 2 行 |
| 答错提示 | 1～2 句 |
| 解析 | 分步展示，每步短句 |

### 6.5 儿童可读性原则

- 避免长句。
- 避免抽象词。
- 避免复杂标点。
- 重要数字加粗或高亮。
- 剧情文本和题目文本分开。
- 题目必须比剧情更清晰。
- 按钮文案必须是明确动作，如“开始调查”“提交答案”“查看线索”。

## 8. 组件规范

## 8.1 按钮组件

### 按钮类型

| 类型 | 用途 | 颜色 |
| --- | --- | --- |
| 主按钮 | 开始调查、提交答案、挑战 Boss | 胡萝卜橙 |
| 次按钮 | 返回地图、查看线索 | 天空蓝或白底描边 |
| 奖励按钮 | 领取勋章、查看奖励 | 徽章金 |
| 成功按钮 | 已完成、继续复习 | 草地绿 |
| 危险按钮 | 放弃、清空记录 | 柔和红，MVP 尽量少用 |
| 禁用按钮 | 未解锁、不可点击 | 浅灰蓝 |

### 按钮尺寸

| 类型 | 高度 | 圆角 | 字号 |
| --- | --- | --- | --- |
| 大按钮 | 52～56px | 20～24px | 18～20px |
| 中按钮 | 44～48px | 16～20px | 16～18px |
| 小按钮 | 36～40px | 12～16px | 14～16px |

### 按钮状态

| 状态 | 表现 |
| --- | --- |
| 默认 | 明亮、清楚 |
| 悬停 | 轻微变亮或上浮 |
| 按下 | 颜色加深或轻微下沉 |
| 禁用 | 灰色、不可点击、可带锁图标 |
| 加载中 | 显示加载图标，避免重复点击 |

### 按钮文案示例

```
开始调查
继续闯关
提交答案
查看线索
挑战 Boss
返回地图
进入悬案馆
领取勋章
```

## 8.2 地图卡片组件

### 用途

用于展示案件入口、地图区域、单元入口。

### 内容结构

```
区域 / 案件插画
案件名称
知识点标签
案件状态
进度条
进入按钮
```

### 状态规范

| 状态 | 视觉表现 |
| --- | --- |
| 未开始 | 明亮正常显示 |
| 进行中 | 显示“调查中”和进度条 |
| 已结案 | 显示“已结案”印章 |
| 未解锁 | 灰度、锁图标、不可点击 |
| 推荐开始 | 轻微高亮或箭头提示 |

### 卡片原则

- 卡片可点击区域要大。
- 入口状态必须清楚。
- 不要让复杂地图细节影响点击识别。
- 移动端卡片优先纵向排列。

## 8.3 导师卡组件

### 用途

用于鹿队长、老师等角色引导孩子。

### 内容结构

```
角色头像
角色名称
对白文本
下一步按钮 / 关闭按钮
```

### 设计规范

| 项目 | 规范 |
| --- | --- |
| 头像 | 单独透明图层 |
| 文本框 | 气泡或档案纸样式 |
| 文本长度 | 每次不超过 2～3 行 |
| 按钮 | 明确下一步 |
| 遮挡 | 不遮挡题目核心区域 |

### 示例文案

```
鹿队长：
小侦探，仓库里的徽章要平均分给 4 个小队。
我们先看看每个小队能分到几枚吧！
```

## 8.4 题目卡组件

### 用途

承载核心答题体验，优先级最高。

### 内容结构

```
关卡名
题号进度
知识点标签
题干
配图，可选
选项 / 输入框
提交按钮
提示按钮
答题反馈
解析区域
```

### 视觉规范

| 项目 | 规范 |
| --- | --- |
| 卡片背景 | 白色或浅奶油色 |
| 边框 | 浅灰蓝或浅橙 |
| 圆角 | 20～28px |
| 阴影 | 轻微柔和阴影 |
| 题干 | 高对比、大字号 |
| 数字 | 可加粗或重点色 |
| 选项 | 大按钮式选项 |
| 反馈 | 明确但温和 |

### 题目状态

| 状态 | 视觉表现 |
| --- | --- |
| 未作答 | 默认卡片 |
| 已选中 | 选项高亮 |
| 答对 | 绿色反馈 + 对勾 |
| 答错 | 柔和红提示 + 加入悬案馆 |
| 显示解析 | 展示分步解析 |
| 已完成 | 显示完成标记 |

### 题目卡禁忌

- 不要把题目放在复杂插画上。
- 不要让装饰元素抢占题目注意力。
- 不要在同屏放过多剧情文字。
- 不要用小字号显示数学题。
- 不要用强烈红色大叉制造挫败感。

## 8.6 线索卡组件

### 用途

承载案件线索和知识线索。

### 类型

| 类型 | 用途 |
| --- | --- |
| 案件线索卡 | 推动剧情、解锁 Boss |
| 知识线索卡 | 复习知识点 |
| 未解锁线索卡 | 激励继续闯关 |

### 知识线索卡结构

```
知识点名称
一句话解释
例题
解题方法
常见错误
关联关卡
掌握状态
```

### 视觉规范

| 项目 | 规范 |
| --- | --- |
| 背景 | 档案纸 / 便签纸风 |
| 颜色 | 奶油黄、浅橙、浅蓝 |
| 装饰 | 回形针、放大镜、印章 |
| 文字区 | 必须干净 |
| 状态 | 已掌握 / 待复习 / 未解锁 |

### 线索卡禁忌

- 不要使用复杂底纹影响阅读。
- 不要把知识点解释写得太长。
- 不要把剧情线索和知识解释混在同一段长文本里。

## 8.7 悬案卡组件

### 用途

展示错题。

### 内容结构

```
错题标题
所属知识点
题目摘要
错误次数
状态：未侦破 / 已侦破
重新调查按钮
```

### 状态视觉

| 状态 | 视觉 |
| --- | --- |
| 未侦破 | 浅红底 + 档案袋图标 |
| 调查中 | 蓝色高亮 |
| 已侦破 | 绿色印章 |
| 多次错误 | 增加“重点调查”标签 |

### 文案原则

错题不叫“错误题目”，而叫：

```
待调查线索
未侦破悬案
重新调查
成功侦破
```

## 8.8 勋章组件

### 用途

展示学习成就。

### 勋章卡结构

```
勋章图标
勋章名称
获得条件
获得时间
获得原因
状态：已获得 / 未获得
```

### 状态视觉

| 状态 | 视觉 |
| --- | --- |
| 已获得 | 彩色、明亮、可点击 |
| 未获得 | 灰色、锁图标 |
| 新获得 | 金色高光、轻微动效 |
| 重点勋章 | 可加金边，后续版本使用 |

### MVP 勋章

| 勋章 | 图形方向 |
| --- | --- |
| 新手小侦探 | 小侦探帽 |
| 线索收集员 | 放大镜 + 线索 |
| 悬案调查员 | 档案袋 |
| 坚持不放弃 | 小脚印 / 小火苗 |
| 破案小能手 | 金色侦探徽章 |

## 8.9 弹窗组件

## 10. 页面级规范

## 10.1 开始页

### 设计目标

让孩子快速知道：

```
我是谁
我要进入哪里
点击哪里开始
```

### 必备元素

- 产品 Logo / 标题
- 动物侦探学院氛围插画
- 昵称输入
- 开始按钮
- 简短引导语

### 禁忌

- 不放复杂产品介绍。
- 不放家长说明长文。
- 不放太多入口。

---

## 10.2 地图页

### 设计目标

地图页是“动物侦探城”的核心入口，要有探索感。

### 必备元素

- 城市地图背景
- 案件入口
- 悬案馆入口
- 线索库入口
- 勋章馆入口
- 当前进度
- 角色引导，可选

### 设计原则

- 地图是氛围，不是复杂导航迷宫。
- 入口必须清楚可点击。
- 未解锁内容必须有锁定状态。
- 已结案内容要有成就标识。

---

## 10.3 案件详情页

### 设计目标

让孩子理解：

```
发生了什么事
我要调查什么
完成后能得到什么
```

### 必备元素

- 案件名称
- 案件背景
- 主要角色
- 调查目标
- 线索进度
- 开始调查按钮

### 文案展示

剧情可以使用导师卡或档案卡。

---

## 10.4 普通关卡页

### 设计目标

专注答题。

### 必备元素

- 关卡名称
- 当前题号
- 题目卡
- 选项 / 输入框
- 提交按钮
- 提示按钮
- 反馈区

### 设计原则

- 题目卡居中。
- 背景弱化。
- 角色引导不要抢占题目。
- 选项按钮足够大。
- 答错反馈温和。

---

## 10.5 线索获得页

### 设计目标

制造小成就感，并推动继续闯关。

### 必备元素

- 获得线索动画或弹窗
- 线索名称
- 简短说明
- 对应知识点
- 继续调查按钮

### 视觉建议

可使用档案卡、印章、放大镜高光。

---

## 10.6 Boss 挑战页

### 设计目标

制造最终挑战感，但不能压迫孩子。

### 必备元素

- Boss 挑战标题
- 已收集线索展示
- 综合题卡
- 提示入口
- 返回线索库入口
- 挑战按钮

### 视觉原则

- 可以比普通关卡更有仪式感。
- 不使用恐怖 Boss 形象。
- Boss 更像“最终谜题”而不是敌人压迫。
- 失败后强调“再调查一次”，不是“失败”。

---

## 10.7 结案页

### 设计目标

总结学习成果，给孩子完成感。

### 必备元素

- 已结案标识
- 案件真相
- 掌握知识点
- 新增勋章
- 错题数量
- 去线索库
- 去悬案馆
- 返回地图

### 视觉建议

使用金色印章、彩带、徽章奖励，但不要遮挡总结信息。

---

## 10.8 悬案馆页

### 设计目标

让错题复习变成“重新调查”。

### 必备元素

- 未侦破列表
- 已侦破列表
- 知识点筛选
- 错误次数
- 重新调查按钮

### 视觉原则

- 神秘但不恐怖。
- 使用档案馆、文件柜、线索板元素。
- 未侦破用柔和红，不制造焦虑。
- 已侦破用绿色印章。

---

## 10.9 线索库页

### 设计目标

让知识复习像查看侦探笔记。

### 必备元素

- 知识卡列表
- 知识点名称
- 掌握状态
- 例题
- 常见错误
- 关联错题

### 视觉原则

- 像侦探笔记 / 档案册。
- 文字区域必须干净。
- 不使用过度复杂背景。

---

## 10.10 勋章馆页

### 设计目标

展示成长和成就。

### 必备元素

- 勋章墙
- 已获得勋章
- 未获得勋章
- 获得时间
- 获得原因

### 视觉原则

- 有收集感。
- 已获得和未获得差异明显。
- 新获得勋章有明显反馈。
- 不做过度竞技排名。

### 背景图可以包含

- 学院场景
- 仓库场景
- 地图道路
- 轻量装饰
- 氛围光影

## 12. 动效规范

### 12.1 动效原则

```
动效服务反馈，不制造干扰。
```

### 12.2 可使用动效

| 场景 | 动效 |
| --- | --- |
| 点击按钮 | 轻微下压 |
| 答对 | 线索闪光、对勾弹出 |
| 答错 | 轻微晃动，不强烈震动 |
| 获得线索 | 卡片翻开 / 印章盖下 |
| 获得勋章 | 勋章放大、金光闪烁 |
| Boss 解锁 | 锁打开、线索汇聚 |
| 结案 | 印章盖下、彩带轻动 |

### 12.3 禁止动效

- 大幅闪烁；
- 高速旋转；
- 强烈震动；
- 长时间无法跳过；
- 影响答题操作；
- 容易让儿童分心的循环动效。

### 12.4 动效时长

| 类型 | 建议时长 |
| --- | --- |
| 按钮反馈 | 100～200ms |
| 卡片出现 | 200～300ms |
| 答对 / 答错反馈 | 300～600ms |
| 获得勋章 | 800～1200ms |
| 页面切换 | 200～400ms |

## 13. 无障碍与儿童友好规范

### 13.1 可点击区域

```
最小点击区域不小于 44px × 44px。
主要按钮建议高度 52px 以上。
```

### 13.2 对比度

- 题干和选项必须高对比。
- 浅色文字不能放在浅色背景上。
- 彩色背景上的文字必须使用深色或白色高对比。

### 13.3 低龄儿童友好

- 关键按钮固定位置，减少寻找成本。
- 每屏只放一个主要操作。
- 错误反馈要鼓励，不批评。
- 状态要图文结合。
- 不依赖复杂手势。
- 不要求快速反应。

### 13.4 数学题展示

- 数字必须清楚。
- 除法符号、余数符号要规范。
- 题目中关键数字可加粗。
- 应用题尽量分行展示。
- 不把复杂题目塞进小卡片。

### 19.1 页面检查

```
页面是否符合童趣警探学院风格？
是否像互动绘本，而不是后台页面？
孩子是否一眼知道主操作？
题目是否足够清晰？
按钮是否足够大？
移动端是否安全？
角色是否没有遮挡核心内容？
背景是否没有影响阅读？
```

### 19.2 组件检查

```
按钮状态是否完整？
题目卡是否清晰？
选项是否容易点击？
线索卡是否适合阅读？
悬案卡是否温和不吓人？
勋章是否有收集感？
导师卡文字是否太长？
```

### 19.3 资产检查

```
背景是否只做场景？
角色是否单独透明图层？
UI 图标是否独立？
文字是否没有写死在图片里？
按钮是否由前端生成？
图片命名是否规范？
是否有源文件？
```

---

## 20. 总结

《动物侦探城》的 UI 设计必须同时服务三类目标：

```
让孩子愿意进入
让孩子看得懂题
让团队能持续扩展
```

因此，本项目 UI 不能只追求漂亮插画，而要建立稳定的视觉系统：

```
统一风格
统一色彩
统一字体
统一组件
统一安全区
统一图片分层
统一交付规范
```

最终目标是形成一个可以长期扩展的儿童探案式学习 Design System，让后续每一个新案件、新关卡、新地图、新勋章都能在统一体验下快速生产和上线。
