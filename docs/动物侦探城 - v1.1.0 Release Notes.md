# 动物侦探城 v1.1.0 Release Notes

## 版本信息

- 产品版本：v1.1.0
- Git tag：v1.1.0，待发布确认后创建
- 发布日期：2026-05-22
- 发布负责人：项目负责人 + 技术负责人
- 对应 Milestone：v1.1.0
- 发布类型：Minor

## 版本目标

将 v1.0.0 的单内容包正式体验版升级为可注册、可选择、可隔离、可校验、可回归的内容扩展底座，为后续切换教材、维护知识体系和建设知识工厂打基础。

## 产品范围

### 本次包含

- 新增内容包 registry。
- 当前除法内容包登记为默认内容包。
- Web 端支持通过 registry 加载内容包。
- Web 端支持通过 `?contentPackageId=...` 指定 bundled 内容包。
- 学习记录按 `questAcademy:progress:{contentPackageId}` 隔离。
- 保留 v1.0.0 / v0.2.0 旧学习记录迁移入口。
- `validate-content` 支持 registry、指定内容包和全量内容包校验。
- `regression-smoke` 支持默认内容包、`--package` 和 `--all`。
- 补齐内容工厂交付清单、审核模板、审核样例和人工门禁。

### 本次不包含

- 不新增正式内容包。
- 不新增课程选择页。
- 不接后端。
- 不做账号系统。
- 不做家长端。
- 不做正式内容后台或 CMS。
- 不改变当前徽章失踪案核心体验。

## 新增

- `content/registry.json` 内容包注册表。
- `ContentPackageRegistry` 和 `ContentPackageRegistryEntry` 类型。
- `getBundledContentRegistry()`、`getBundledContentPackageEntry()` 和按内容包加载 runtime 的接口。
- 按内容包隔离的 localStorage key。
- `validate-content` CLI 参数：
  - `--package <contentPackageId>`
  - `--all`
  - `--registry <path>`
- `test:regression` CLI 参数：
  - `--package <contentPackageId>`
  - `--all`
- 内容工厂交付清单与审核样例文档。

## 优化

- 默认内容包不再直接由调用方硬编码，改由 registry 决定。
- 学习记录不会被未来不同内容包互相覆盖。
- 内容校验和浏览器回归可以对准指定内容包运行。
- 缺失内容包会在回归脚本启动浏览器前清晰失败。
- 发布前文档补齐内容工厂的 draft -> approved -> published 人工门禁。

## 修复

- 修正多内容包扩展前的存储污染风险。
- 修正回归脚本只能验证单一硬编码内容包的问题。
- 补齐 registry entryPath、默认包唯一性和缺失包的校验覆盖。

## 已知问题

- 暂无 P0 / P1 已知问题。

## 是否影响学习记录

- 是否影响 localStorage：是。
- 是否需要数据迁移：是，旧 key `animalDetectiveCityMvp` 仍作为迁移来源。
- 新 key：`questAcademy:progress:{contentPackageId}`。
- 数据版本：`dataVersion` 仍为 `1.0.0`，本版本不改变学习记录结构主版本。
- 是否存在数据丢失风险：当前设计为新 key 优先、旧 key 迁移来源；不同内容包学习记录互不覆盖。

## 回滚方案

- 回滚产品版本：回滚到 tag `v1.0.0`。
- 内容包版本：继续使用 `math.bsd.g2.s2.unit-1-division@0.1.0`。
- localStorage：v1.1.0 会写入包级 key；回滚到 v1.0.0 后旧版仍读取旧 key。必要时可提示用户重新开始或保留 v1.1.0 后再恢复。
- 技术回滚：撤回 v1.1.0 Release，创建 patch 分支修复，或将主分支回滚到 `v1.0.0` tag 后重新发布。
